## 🔴 问题根源

### 推测 1: Transaction 处于"僵死"状态

**可能原因**：
- 之前的购买成功但没有完成
- 清理模式时 `completePurchase()` 失败
- StoreKit 内部状态不一致

**证据**：
1. StoreKit 报告有 pending transaction（所以报错）
2. 但 `restorePurchases()` 检测不到（没有回调）
3. 清理模式也检测不到（超时）

### 推测 2: StoreKit 缓存问题

**可能原因**：
- StoreKit 的本地缓存没有更新
- Transaction 已经在服务端完成
- 但本地还认为是 pending

**证据**：
```
596: 清理模式：开始
599: 清理超时，可能没有待处理交易
600: 清理模式：完成
```
清理模式超时，说明没有检测到任何交易。

### 推测 3: 时序问题

**可能原因**：
- 清理模式和实际购买之间有时间差
- StoreKit 正在处理之前的交易
- 但状态还没有更新

**证据**：
- 每次购买都立即遇到这个错误
- 说明交易一直处于 pending 状态

### 方案 1: 多次尝试恢复购买 ✅ 已实现


## 🔧 技术限制

### Flutter in_app_purchase 包的限制

**无法直接访问**：
1. `SKPaymentQueue.transactions` - 查看所有 pending transactions
2. `SKPaymentQueue.finishTransaction()` - 强制完成指定交易
3. StoreKit 的内部状态

**只能使用**：
1. `buyConsumable()` - 发起购买
2. `restorePurchases()` - 触发恢复购买
3. `completePurchase()` - 完成已验证的交易

### StoreKit 的限制

**StoreKit 的规则**：
1. 同一产品同时只能有一个 pending transaction
2. 必须完成当前交易才能开始新交易
3. Transaction 状态由 Apple 服务器控制

**无法强制**：
1. 清除特定的 pending transaction
2. 重置 StoreKit 状态
3. 跳过验证直接完成交易

## 🎯 为什么会出现这个问题？

### 根本原因

**优化前的流程问题**：
```
购买成功
    ↓
开始验证
    ↓
验证失败/应用崩溃 ❌
    ↓
收据丢失 ❌
    ↓
交易没有完成 ❌
    ↓
变成 pending transaction ❌
```

**现在的优化**：
```
购买成功
    ↓
立即保存收据 ✅（新增）
    ↓
开始验证
    ↓
验证成功 → 删除缓存 → 完成交易 ✅
验证失败 → 保留缓存 → 完成交易 ✅
    ↓
应用重启 → 自动补发 ✅
```

**效果**：
- 以后不应该再产生新的 pending transaction
- 现有的问题需要手动重启解决
