# 调试工具开关使用说明

## 📋 功能概述

为了方便截图演示，添加了全局调试工具开关，可以隐藏所有调试框架（如网络调试悬浮窗）。

---

## 🎯 核心功能

### 1. 全局调试工具开关

- **位置**: 设置页面 > 🔧 开发者工具
- **功能**: 一键隐藏/显示所有调试框架
- **使用场景**: 
  - ✅ 截图演示时隐藏调试信息
  - ✅ 录屏时保持界面干净
  - ✅ 向客户展示时隐藏开发工具

## 🚀 使用方式

### 方式 1: 通过设置页面（推荐）

1. 打开 **设置页面**
2. 滚动到底部找到 **🔧 开发者工具** 区域（仅 Debug 模式显示）
3. 切换 **调试框架显示** 开关

### 方式 2: 通过网络调试悬浮窗

1. 点击左上角的网络调试悬浮按钮（黑色圆形按钮）
2. 在展开的列表顶部，点击 **绿色眼睛图标** 👁️
3. 所有调试框架立即隐藏

### 方式 3: 代码中切换

```dart
// 导入配置
import 'package:yakaixin_app/core/config/debug_config.dart';

// 通过 Provider 切换
ref.read(debugToolsEnabledProvider.notifier).state = false; // 隐藏
ref.read(debugToolsEnabledProvider.notifier).state = true;  // 显示

// 在任何 ConsumerWidget 中监听
final isDebugEnabled = ref.watch(debugToolsEnabledProvider);
```

## 🔧 技术实现

### 1. 配置文件

**文件**: `lib/core/config/debug_config.dart`

```dart
/// 调试工具开关状态 Provider
final debugToolsEnabledProvider = StateProvider<bool>((ref) => true);
```

### 2. 网络调试悬浮窗集成

**文件**: `lib/core/widgets/network_debug_overlay.dart`

```dart
@override
Widget build(BuildContext context) {
  final logs = ref.watch(networkLogsProvider);
  final isMockEnabled = ref.watch(mockEnabledProvider);
  // ✅ 监听调试工具全局开关
  final isDebugEnabled = ref.watch(debugToolsEnabledProvider);

  // 在 Debug 模式下才显示悬浮按钮
  if (!kDebugMode) {
    return widget.child;
  }
  
  // ✅ 如果调试工具被关闭，只显示主内容
  if (!isDebugEnabled) {
    return widget.child;
  }

  return Stack(
    children: [
      widget.child,
      // ... 调试界面
    ],
  );
}
```

            ref.read(debugToolsEnabledProvider.notifier).state = value;

  ),

## 📌 关键特性

### ✅ 响应式更新

- 使用 Riverpod `StateProvider` 管理状态
- 所有监听该状态的 Widget 自动更新
- 切换开关后，调试框架立即显示/隐藏

### ✅ 仅 Debug 模式可见

- 设置页面的开关仅在 `kDebugMode` 下显示
- Release 版本不会显示开发者工具区域
- 不影响生产环境

### ✅ 自动关闭所有弹窗

当点击网络调试悬浮窗的眼睛图标时：

```dart
void _toggleDebugTools() {
  final currentState = ref.read(debugToolsEnabledProvider);
  ref.read(debugToolsEnabledProvider.notifier).state = !currentState;
  
  // 关闭所有弹窗
  setState(() {
    _isExpanded = false;
    _showingEnvSelector = false;
    _selectedLog = null;
  });
}
```

## 🔍 扩展能力

### 添加更多调试工具

如果未来需要添加更多调试工具，只需：

1. 在新工具的 Widget 中监听 `debugToolsEnabledProvider`
2. 根据状态决定是否显示

**示例**:

```dart
class MyDebugTool extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final isDebugEnabled = ref.watch(debugToolsEnabledProvider);
    
    // 调试工具关闭时不显示
    if (!isDebugEnabled) {
      return const SizedBox.shrink();
    }
    
    return Container(
      // ... 调试工具 UI
    );
  }
}
```

### 添加更多开关

在设置页面的 **🔧 开发者工具** 区域添加更多开关：

```dart
_buildDebugSwitchItem(
  title: '性能监控',
  subtitle: '显示FPS和内存使用',
  value: ref.watch(performanceMonitorProvider),
  onChanged: (value) {
    ref.read(performanceMonitorProvider.notifier).state = value;
  },
),
```

## ⚠️ 注意事项

1. **仅 Debug 模式有效**
   - Release 版本不会显示任何调试工具
   - 不影响生产环境性能

2. **状态不持久化**
   - 应用重启后恢复默认值（显示）
   - 如需持久化，可结合 SharedPreferences

3. **完全隐藏**
   - 关闭后，调试框架完全不渲染
   - 不仅是视觉隐藏，还节省性能
