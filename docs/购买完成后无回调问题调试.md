# 购买完成后无回调问题调试

**更新时间**: 2025-01-25  
**问题**: 用户输入密码点击"好"完成购买后，没有触发回调，UI 一直显示"正在支付..."

---

## 🔍 问题分析

### 当前日志

```
flutter: 🍎 ========== iOS内购支付 ==========
flutter:    💡 iOS内购是异步模式，需要等待回调
flutter: 开始购买 - 订单:597673870069076300
flutter: 产品ID: com.yakaixin.yakaixin.1 (基于价格: ¥1.0)
flutter: 查询产品: {com.yakaixin.yakaixin.1}
flutter:    ⏳ 等待支付结果...
flutter: 找到 1 个产品
flutter: 产品: 一元课程 - ¥1
flutter: 💰 消耗型商品，允许重复购买
flutter: 购买请求已发送，等待用户确认

← 用户输入密码，点击"好"
← 应该触发 _onPurchaseUpdate 回调
← 但没有任何日志！
```

### 可能的原因

1. **清理模式未正确结束**
   ```dart
   if (_isCleaningMode) {
     // ⚠️ 如果清理模式未正确结束
     // 真正的购买回调会被 continue 跳过
     continue;
   }
   ```

2. **purchaseStream 监听未生效**
   ```dart
   _subscription = _iap.purchaseStream.listen(
     _onPurchaseUpdate,
     // ⚠️ 可能监听设置有问题
   );
   ```

3. **StoreKit 回调丢失**
   - 沙盒环境不稳定
   - 回调确实没有触发

### 第 3 步：如果没有看到日志

**说明回调根本没触发**，可能是以下情况：

#### 情况 A：清理模式未结束

如果看到：
```

🧹 清理模式：直接完成交易
   ⏭️ 跳过此交易（清理模式）

**解决方案**：
1. 重启应用
2. 等清理模式完成后（5秒）再购买
3. 查看启动日志确认清理模式已结束：
   ```

   清理超时，可能没有待处理交易
   清理模式：完成
   ```

#### 情况 B：purchaseStream 未监听

查看启动日志，确认以下内容：
```
flutter: 内购服务初始化成功
```

如果没有这个日志，说明 `initialize()` 未执行。

#### 情况 C：StoreKit 回调丢失

如果：
- ✅ 清理模式已完成
- ✅ 初始化成功
- ✅ 但仍然没有回调

**这是 StoreKit 沙盒环境 Bug**，解决方案：

1. **重启应用**
   ```
   - 关闭 App
   - 重新打开
   - 收据缓存机制会自动验证
   ```

2. **手动清理 Pending**
   ```
   - 进入设置页
   - 点击"🧹 清理未完成交易"
   - 查看日志
   ```

3. **换沙盒账号**
   ```
   - 退出当前沙盒账号

   - 重新登录

## 🐛 已知问题

### 问题 1: 清理模式耗时过长

**现象**：
- 清理模式开始后等待 5 秒
- 用户在这 5 秒内购买
- 购买回调被清理模式拦截

**临时解决方案**：
- 等待应用启动 5 秒后再购买
- 或在购买前检查 `_isCleaningMode` 状态

**永久解决方案**（后续优化）：
```dart
// 购买前检查清理模式
Future<void> purchase(...) async {
  // ✅ 如果正在清理，等待清理完成
  if (_isCleaningMode && _cleaningCompleter != null) {
    debugPrint('等待清理模式完成...');
    await _cleaningCompleter!.future;
  }
  
  // 开始购买
  // ...
}
```

### 问题 2: StoreKit 沙盒回调丢失

**现象**：
- 用户完成支付
- 但 purchaseStream 没有任何回调
- 这是 Apple 沙盒环境已知问题

**临时解决方案**：
1. 重启应用（收据缓存会自动验证）
2. 手动清理 Pending
3. 换沙盒账号

**保护机制**：
- ✅ 收据缓存：支付后立即保存
- ✅ 自动补发：启动时自动验证
- ✅ 用户指引：提示重启应用
