### 方法2: 从日志文件中提取

#### 步骤1: 保存完整日志

```bash
# 运行应用并保存日志
flutter run 2>&1 | tee ios_receipt_log.txt
```

#### 步骤2: 使用脚本提取票据

创建提取脚本 `extract_receipt.sh`:

```bash
#!/bin/bash
# 提取票据数据的脚本

LOG_FILE="ios_receipt_log.txt"
OUTPUT_FILE="receipt_data.txt"

# 提取 RECEIPT_DATA_START 和 RECEIPT_DATA_END 之间的内容
sed -n '/RECEIPT_DATA_START/,/RECEIPT_DATA_END/p' "$LOG_FILE" | \
  sed '1d;$d' > "$OUTPUT_FILE"

echo "✅ 票据数据已提取到 $OUTPUT_FILE"
echo "📋 数据长度: $(wc -c < $OUTPUT_FILE) 字节"
```

运行脚本:

```bash
chmod +x extract_receipt.sh
./extract_receipt.sh
```

#### 步骤3: 查看提取结果

```bash
cat receipt_data.txt
```

### 方法3: 使用 Python 脚本提取

创建 `extract_receipt.py`:

```python
#!/usr/bin/env python3
import re

def extract_receipt(log_file):
    """从日志文件中提取票据数据"""
    with open(log_file, 'r', encoding='utf-8') as f:
        content = f.read()
    
    # 使用正则表达式提取 RECEIPT_DATA_START 和 RECEIPT_DATA_END 之间的内容
    pattern = r'RECEIPT_DATA_START\n(.*?)\nRECEIPT_DATA_END'
    matches = re.findall(pattern, content, re.DOTALL)
    
    if matches:
        receipt_data = matches[0].strip()
        
        # 保存到文件
        with open('receipt_data.txt', 'w', encoding='utf-8') as f:
            f.write(receipt_data)
        
        print(f"✅ 票据数据已提取到 receipt_data.txt")
        print(f"📋 数据长度: {len(receipt_data)} 字符")
        print(f"📦 数据预览: {receipt_data[:100]}...")
        
        return receipt_data
    else:
        print("❌ 未找到票据数据")
        return None

if __name__ == '__main__':
    import sys
    
    if len(sys.argv) < 2:
        print("用法: python3 extract_receipt.py <日志文件>")
        print("示例: python3 extract_receipt.py ios_receipt_log.txt")
        sys.exit(1)
    
    log_file = sys.argv[1]
    extract_receipt(log_file)
```

运行脚本:

```bash
# 先保存日志
flutter run 2>&1 | tee ios_receipt_log.txt

# 提取票据
python3 extract_receipt.py ios_receipt_log.txt
```

## 🔍 验证票据数据

### 检查数据格式

票据数据应该满足以下特征：

1. **Base64 编码格式**
   - ✅ 只包含字符：`A-Z`, `a-z`, `0-9`, `+`, `/`, `=`
   - ✅ 长度通常在 2000-5000 字符
   - ✅ 可能以 `MII` 开头（常见）

2. **数据长度**
   ```bash
   # 检查字符数
   echo -n "YOUR_RECEIPT_DATA" | wc -c
   
   # 应该在 2000-5000 之间
   ```

3. **Base64 解码测试**（可选）
   ```bash
   # macOS/Linux
   echo "YOUR_RECEIPT_DATA" | base64 -d | file -
   
   # 应该显示类似：data
   ```

## ⚠️ 常见问题

### Q1: 找不到 RECEIPT_DATA_START

**原因**:
- 没有触发购买流程
- 清理模式没有待处理交易
- 票据获取失败

**解决**:
1. 确保应用中有未完成的交易
2. 或者完成一次真实购买
3. 查看日志是否有错误

### Q2: 票据数据被截断

**不可能！**现在的实现确保完整输出：
- ✅ 使用 `print(receiptData)` 完整打印
- ✅ 不使用任何截断操作
- ✅ 有明确的开始和结束标记

### Q3: 日志太长，不好找

**解决方法**:

1. **过滤日志**
   ```bash
   flutter run 2>&1 | grep -A 200 "RECEIPT_DATA_START"
   ```

2. **保存日志文件**
   ```bash
   flutter run 2>&1 | tee full_log.txt
   # 然后在文本编辑器中搜索
   ```

3. **使用提取脚本**（见上方方法2、方法3）

### Q4: 复制时多了空格或换行

**正确操作**:
- ✅ 只选中 Base64 字符串部分
- ✅ 不要包含 `RECEIPT_DATA_START` 行
- ✅ 不要包含 `RECEIPT_DATA_END` 行
- ✅ 不要添加任何额外字符

**验证方法**:
```bash
# 复制后，粘贴到文本编辑器
# 检查开头和结尾是否只有 Base64 字符
# 没有多余的空格、换行、标记
```
