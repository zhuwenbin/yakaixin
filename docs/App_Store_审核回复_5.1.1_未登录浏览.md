# App Store 审核回复 - Guideline 5.1.1

## 主题：关于未登录用户访问功能的说明

尊敬的审核团队：

感谢您的反馈。我们已经仔细审查了 Guideline 5.1.1 的要求，并确认我们的应用已经支持未登录用户自由浏览大部分内容。

## 应用已支持的功能

### ✅ 未登录用户可以自由访问的功能

我们的应用允许未登录用户（游客模式）自由访问以下功能，**无需注册或登录**：

#### 1. **首页浏览**
- **路径**：应用启动后直接进入首页
- **功能**：可以浏览所有商品列表，包括：
  - 题库商品（试卷、模考、章节练习）
  - 网课商品
  - 直播商品
  - 秒杀推荐商品
- **显示内容**：商品名称、价格、封面图片、商品描述等完整信息

#### 2. **商品详情页查看**
- **路径**：首页 → 点击任意商品 → 商品详情页
- **功能**：可以查看商品的完整详情，包括：
  - 商品介绍
  - 价格信息
  - 课程大纲（网课）
  - 题目数量（题库）
  - 有效期信息
- **说明**：未登录用户可以完整浏览所有商品信息

#### 3. **题库页面浏览**
- **路径**：底部导航栏 → "题库" Tab
- **功能**：可以浏览题库首页，包括：
  - 每日一测商品
  - 技能模拟商品
  - 章节练习商品
  - 商品列表和价格信息

#### 4. **课程页面浏览**
- **路径**：底部导航栏 → "课程" Tab
- **功能**：可以浏览课程列表，包括：
  - 网课列表
  - 直播列表
  - 课程详情和价格信息

#### 5. **专业选择**
- **路径**：首页顶部 → 专业选择器
- **功能**：未登录用户可以选择不同的专业，浏览对应专业的商品内容

#### 6. **其他公开页面**
- 隐私政策
- 用户服务协议
- 关于我们
- 设置页面（部分功能）

### ⚠️ 需要登录的功能（账户相关功能）

以下功能需要用户登录，因为这些功能与用户账户直接相关：

1. **购买商品** - 需要账户来管理订单和支付记录
2. **开始练习/学习** - 需要账户来保存学习进度和答题记录
3. **打卡功能** - 需要账户来记录打卡数据
4. **个人中心** - 需要账户来查看个人信息、学习统计、订单记录等
5. **收藏功能** - 需要账户来保存收藏内容
6. **错题本** - 需要账户来保存错题记录

## 用户体验说明

### 未登录用户的体验流程

1. **应用启动** → 直接进入首页，无需登录
2. **浏览商品** → 可以自由浏览所有商品列表和详情
3. **查看价格** → 所有商品的价格信息都完整显示
4. **尝试购买** → 点击"购买"或"开始练习"时，会弹出友好的登录提示
5. **登录提示** → 提示用户"此操作需要登录账户，请先登录后再试"，并提供"去登录"和"取消"选项
6. **登录后** → 用户可以继续之前的操作（购买、练习等）

### 登录提示的设计

- **友好的提示**：使用统一的确认对话框，说明需要登录的原因
- **非强制**：用户可以选择"取消"，继续浏览其他内容
- **便捷的登录**：提供多种登录方式（手机号验证码登录、密码登录、微信登录）

## 技术实现说明

### 路由配置

我们的应用使用路由拦截机制，将页面分为两类：

1. **公开页面**（无需登录）：
   - 首页（`/main-tab`）
   - 商品详情页（`/goods-detail`）
   - 课程详情页（`/course-goods-detail`）
   - 题库页面（通过 Tab 导航访问）
   - 课程页面（通过 Tab 导航访问）
   - 其他设置、协议页面

2. **需要登录的页面**：
   - 个人中心（部分功能）
   - 订单页面
   - 学习记录页面

### 数据加载策略

- **未登录状态**：所有公开数据正常加载，需要登录的数据静默处理（不显示错误提示）
- **登录状态**：所有数据正常加载，包括用户专属数据

## 符合 Guideline 5.1.1 的要求

根据 Guideline 5.1.1 的要求：

> "Apps may not require users to enter personal information to function, except when directly relevant to the core functionality of the app or required by law."

我们的应用：
- ✅ **允许未登录用户浏览内容**：用户可以在不提供任何个人信息的情况下，浏览所有商品、查看价格、了解课程内容
- ✅ **只在必要时要求登录**：只有在用户想要购买、开始学习等账户相关操作时，才要求登录
- ✅ **提供清晰的登录提示**：明确说明为什么需要登录，并允许用户选择不登录继续浏览
