# 购买前等待清理模式完成 - 修复竞态条件

**更新时间**: 2025-01-25  
**问题**: 用户购买时，清理模式可能还未结束，导致回调被跳过  
**解决方案**: 购买前主动等待清理模式完成

---

## 🔴 问题分析

### 原始流程

```
0秒  应用启动
0秒  initialize() → _clearPendingTransactions()
0秒  _isCleaningMode = true
0-5秒 等待清理（5秒超时）
5秒  _isCleaningMode = false

用户操作：
5秒  看到界面
6秒  点击购买 ← 这时 _isCleaningMode = false ✅
7秒  输入密码，点击"好"
8秒  回调触发 → 正常处理 ✅
```

### 竞态条件场景

```
0秒  应用启动
0秒  initialize() → _clearPendingTransactions()
0秒  _isCleaningMode = true

用户操作：
2秒  用户快速操作，点击购买 ← ❌ _isCleaningMode = true
2秒  purchase() 执行
3秒  发送购买请求
4秒  用户输入密码，点击"好"
5秒  回调触发 → ❌ 但 _isCleaningMode 可能仍是 true
5秒  回调被跳过！

if (_isCleaningMode) {
  // ❌ 跳过了真正的购买回调
  continue;
}
```

### 为什么会发生？

1. **清理模式耗时 5 秒**
2. **用户可能在 2-3 秒内就完成购买流程**
3. **回调可能在清理模式结束前到达**

---

## ✅ 解决方案

### 修改点

在 `purchase()` 方法开始时，添加等待逻辑：

```dart
Future<void> purchase({
  required String orderId,
  required String goodsId,
  required double amount,
  required String financeBodyId,
  required String studentId,
  String? goodsName,
  required IAPCallback callback,
}) async {
  try {
    // ✅ 等待清理模式完成（避免竞态条件）
    if (_isCleaningMode && _cleaningCompleter != null) {
      debugPrint('⏳ 检测到清理模式进行中，等待完成...');
      
      try {
        // 最多等待 6 秒（清理模式超时是 5 秒）
        await _cleaningCompleter!.future.timeout(const Duration(seconds: 6));
        debugPrint('✅ 清理模式已完成，继续购买');
      } catch (e) {
        debugPrint('⚠️ 等待清理模式超时，继续购买');
      }
    }

    debugPrint('开始购买 - 订单:$orderId 商品:$goodsId 价格:¥$amount');
    
    // ... 继续购买逻辑
  }
}
```

### 工作原理

```
场景 1: 清理模式已完成
─────────────────────────
用户点击购买
  ↓
检查 _isCleaningMode
  ↓
_isCleaningMode = false
  ↓
直接继续购买 ✅
```

```
场景 2: 清理模式进行中
─────────────────────────
用户点击购买
  ↓
检查 _isCleaningMode
  ↓
_isCleaningMode = true
  ↓
等待 _cleaningCompleter.future
  ↓
清理模式完成
  ↓
_isCleaningMode = false
  ↓
继续购买 ✅
```

### 时间线对比

**修改前**：
```
0s   清理开始
2s   用户购买 ← 不等待
3s   购买请求发出
5s   回调到达 ← ❌ 清理模式还在，被跳过
5s   清理结束（太晚了）
```

**修改后**：
```
0s   清理开始
2s   用户购买 ← 检测到清理模式
2s   等待清理完成...
5s   清理结束
5s   继续购买 ✅
6s   购买请求发出
8s   回调到达 ← ✅ 清理已完成，正常处理
```

## 💡 为什么是 6 秒超时？

```dart
await _cleaningCompleter!.future.timeout(const Duration(seconds: 6));
```

**理由**：
1. 清理模式本身超时是 5 秒
2. 加 1 秒缓冲，确保清理模式一定完成
3. 避免无限等待（如果 _cleaningCompleter 有 bug）

**极端情况**：
- 清理模式 5 秒超时
- 购买等待 6 秒超时
- 总计最多 11 秒（几乎不可能）
