# 支付成功页背景图片路径修复

## 问题描述

用户反馈：参考小程序，支付成功页面的背景图片路径不正确。

## 问题原因

### 小程序实现（正确）

在小程序 `paySuccess.vue` 中，所有图片都使用**完整的URL**：

**Line 6-8: 成功图标**
```vue
<image src="https://xy-shunshun-pro.oss-cn-hangzhou.aliyuncs.com/public/26e2174185623886722986_%E7%BC%96%E7%BB%84%2013%402x.png">
</image>
```

**Line 192: 二维码背景框**
```css
.qrcode {
  background-image: url(https://xy-shunshun-pro.oss-cn-hangzhou.aliyuncs.com/public/46c2174185637060211201_%E5%BD%A2%E7%8A%B6%E7%BB%93%E5%90%88%402x.png);
}
```

### Flutter实现（错误）

在 Flutter `pay_success_page.dart` 中，使用了**相对路径**：

**Line 182（修改前）：成功图标**
```dart
child: Image.network(
  ApiConfig.completeImageUrl('public/26e2174185623886722986_%E7%BC%96%E7%BB%84%2013%402x.png'),
  // ❌ 相对路径，可能拼接错误
)
```

**Line 218（修改前）：二维码背景框**
```dart
image: DecorationImage(
  image: NetworkImage(
    ApiConfig.completeImageUrl('public/46c2174185637060211201_%E5%BD%A2%E7%8A%B6%E7%BB%93%E5%90%88%402x.png'),
    // ❌ 相对路径，可能拼接错误
  ),
)
```

### 问题分析

1. **`ApiConfig.completeImageUrl()` 可能拼接错误**
   - 如果baseUrl配置不正确
   - 如果路径拼接逻辑有问题
   - 导致图片无法加载

2. **与小程序不一致**
   - 小程序直接使用完整URL，稳定可靠
   - Flutter使用相对路径，增加了不确定性

## 注意事项

### 1. URL编码问题

小程序和Flutter中的中文字符都需要正确编码：

```
原始：编组 13@2x.png
编码：%E7%BC%96%E7%BB%84%2013%402x.png
```

### 2. 缓存问题

网络图片会被缓存，修改URL后需要：
- ✅ 清除应用缓存
- ✅ 重启应用
- ✅ 或使用缓存清除工具

### 3. 图片格式

- ✅ 使用 PNG 格式（支持透明）
- ✅ 使用 @2x 高清图
- ✅ 确保服务器正确配置 MIME 类型
