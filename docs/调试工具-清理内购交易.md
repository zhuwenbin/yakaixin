**位置**: 设置页面 > 🔧 开发者工具 > 清理未完成交易  
**功能**: 手动清理 iOS 内购的 pending transaction  
**适用**: Debug 模式 + iOS 平台

---

## 🎯 功能说明

### 什么是 Pending Transaction？

**定义**：
- 用户已支付成功
- 但交易未完成 (未调用 `completePurchase`)
- StoreKit 认为交易还在进行中

**问题表现**：
```
购买时报错：
❌ PlatformException(storekit_duplicate_product_object, ...)
❌ 错误: 检测到未完成的交易，但无法自动处理
```

```
    ↓
```

## ⚠️ 注意事项

### 1. 操作时长

**清理需要约 10 秒**：
- 3 次恢复购买
- 每次等待 3 秒
- 总共约 10 秒

**期间请勿**：
- ❌ 关闭应用
- ❌ 切换页面
- ❌ 锁屏
- ❌ 点击其他按钮

### 2. 网络要求

**需要网络连接**：
- ✅ 恢复购买需要连接 App Store
- ✅ 验证收据需要连接后台服务器
- ❌ 无网络时无法清理

### 3. 清理效果

**可能的结果**：

| 情况 | 日志表现 | 结果 | 下一步 |
|------|---------|------|--------|
| 成功清理 | "收到购买更新" + "验证成功" | ✅ 清理完成 | 可以正常购买 |
| 无交易 | 无回调 | ✅ 没有 pending transaction | 可以正常购买 |
| 清理失败 | 有回调但验证失败 | ⚠️ 收据已保存 | 重启应用补发 |
| 无法处理 | 无回调，仍报错 | ❌ 交易僵死 | 需要手动清理 |

### 4. 失败后的处理

**如果清理无效**：

```

   - 最简单
   - 成功率 70%

方案 2: 退出登录 App Store
   - 设置 → App Store → 退出登录
   - 重新登录
   - 重启应用
   - 成功率 90%

方案 3: 删除应用重装
   - 删除应用
   - Xcode 重新运行
   - 成功率 95%

   - 使用新测试账号
   - 成功率 100%
```

### IAPService 方法

```dart
/// 🧹 手动清理所有 pending transactions
Future<void> clearAllPendingTransactions() async {
  // 1. 第一次恢复购买
  await _iap.restorePurchases();
  await Future.delayed(const Duration(seconds: 3));
  
  // 2. 第二次恢复购买
  await _iap.restorePurchases();
  await Future.delayed(const Duration(seconds: 3));
  
  // 3. 第三次恢复购买
  await _iap.restorePurchases();
  await Future.delayed(const Duration(seconds: 3));
}
```

## 📋 常见问题

### Q1: 为什么要尝试 3 次？

**答案**：
- 第一次可能触发 StoreKit 刷新状态
- 第二次可能检测到刷新后的交易
- 第三次确保没有遗漏
- 增加成功率

### Q2: 清理会影响已购买的内容吗？

**答案**：不会
- 只处理未完成的交易
- 不影响已验证的购买
- 不会删除用户权限

### Q3: 为什么有时候没有效果？

**原因**：
- 交易可能处于"僵死"状态
- StoreKit 内部状态不一致
- 需要重启应用来刷新 StoreKit

**解决**：
- 重启应用
- 或退出登录 App Store

### Q4: 生产环境用户需要这个功能吗？

**答案**：不需要
- 这是开发调试工具
- 仅在 Debug 模式显示
- 生产环境有自动补发机制
