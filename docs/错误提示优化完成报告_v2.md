#### 1.2 API拦截器优化

**文件**: `lib/core/network/api_interceptor.dart`

**优化内容**:
- ✅ **msg提取增强**: `_extractErrorMessage()` 方法现在支持：

**代码示例**:
```dart
/// 提取错误消息
/// 示例:
/// - msg: "账号密码不匹配" → "账号密码不匹配"
/// - msg: ["手机号格式错误", "验证码错误"] → "手机号格式错误, 验证码错误"
/// - msg: null → "哎呦,出错啦~~"
/// - msg: "" → "哎呦,出错啦~~"
String _extractErrorMessage(dynamic msg) {
  if (msg == null) return '哎呦,出错啦~~';
  
  if (msg is String) {
    return msg.trim().isEmpty ? '哎呦,出错啦~~' : msg.trim();
  }
  
  if (msg is List) {
    // ✅ 过滤空字符串后拼接
    final nonEmptyMessages = msg
        .where((item) => item != null && item.toString().trim().isNotEmpty)
        .map((item) => item.toString().trim())
        .toList();
    
    if (nonEmptyMessages.isEmpty) return '哎呦,出错啦~~';
    
    return nonEmptyMessages.join(', ');
  }
  
  // ✅ 其他类型转为字符串
  final strMsg = msg.toString().trim();
  return strMsg.isEmpty ? '哎呦,出错啦~~' : strMsg;
}
```

## 🎯 标准错误处理模式

### Provider 层标准代码

```dart
Future<void> loadData() async {
  try {
    state = state.copyWith(isLoading: true, error: null);
    
    // 调用 Service
    final result = await _service.getData();
    
    state = state.copyWith(
      data: result,
      isLoading: false,
    );
    
  } on DioException catch (e) {
    // ✅ DioException: 使用拦截器已处理的友好信息
    final errorMsg = e.error?.toString() ?? '操作失败，请稍后重试';
    state = state.copyWith(
      isLoading: false,
      error: errorMsg,
    );
    
  } on Exception catch (e) {
    // ✅ Service层的Exception: 使用ErrorMessageMapper
    final errorMsg = ErrorMessageMapper.mapException(e);
    state = state.copyWith(
      isLoading: false,
      error: errorMsg,
    );
    
  } catch (e) {
    // ✅ 其他异常: 使用兜底提示
    state = state.copyWith(
      isLoading: false,
      error: '操作失败，请稍后重试',
    );
    print('❌ [未预期错误] $e');
  }
}
```

## 📌 注意事项

### ✅ DO - 应该做的

1. **在Provider中捕获异常时**:
   - 优先捕获 `DioException`（使用 `e.error?.toString()`）
   - 其次捕获 `Exception`（使用 `ErrorMessageMapper.mapException(e)`）
   - 最后兜底 `catch (e)`（显示默认错误提示）

2. **在Service中抛出异常时**:
   - 抛出有意义的 `Exception('具体错误描述')`
   - DioException 直接 `rethrow`，交给拦截器处理

3. **显示错误提示时**:
   - 使用 `EasyLoading.showError(errorMsg)` 或
   - 使用 `ref.listen()` 监听 `state.error` 并显示

### ❌ DON'T - 不应该做的

1. ❌ **不要使用**:
   - `e.toString()` (在错误处理中)
   - `e.message` (DioException.message 可能为 null)
   - 单独的 `catch (e) { error: e.toString() }`

2. ❌ **不要显示技术错误**:
   - 任何包含 `Exception:` 前缀的信息
   - 任何包含技术关键词的信息

3. ❌ **不要忽略兜底处理**:
   - 所有 async 方法必须有 try-catch
   - 所有 catch 块必须有最后的兜底 `catch (e)`
