# 鸿蒙系统视频播放优化方案

## 参考资源
- 博客：https://blog.csdn.net/weixin_64166120/article/details/156055778
- 目标：解决鸿蒙系统（HarmonyOS）上的视频播放问题

## 问题分析

### 常见问题
1. **硬件解码器不兼容**：鸿蒙系统的硬件解码器可能不支持某些视频编码格式（H.264/H.265）
2. **MediaCodec 错误**：`failed to allocate buffers`、`insufficient resources` 等
3. **高分辨率视频播放失败**：如 2560x1440 等分辨率超出硬件解码能力

### 根本原因
- 鸿蒙系统采用自主设计的微内核架构，与 Android 的 MediaCodec 实现存在差异
- 硬件解码器资源限制，无法处理高分辨率视频
- ExoPlayer 在鸿蒙系统上的兼容性问题

## 解决方案

### 1. 更新 video_player 到最新版本

```yaml
# pubspec.yaml
dependencies:
  video_player: ^2.9.1  # ✅ 从 2.8.2 升级到 2.9.1
  chewie: ^1.7.5
```

**优势**：
- 最新版本包含对鸿蒙系统的兼容性改进
- 更好的错误处理和重试机制
- 优化的 ExoPlayer 配置

### 2. 创建 ExoPlayer 配置（ExoPlayerConfig.kt）

**位置**：`android/app/src/main/kotlin/com/yakaixin/yakaixin/android/ExoPlayerConfig.kt`

**功能**：
- 检测鸿蒙系统
- 为鸿蒙系统创建优化的 `RenderersFactory`
- 优先使用软件解码器，避免硬件解码器兼容性问题
- 启用解码器回退机制（硬件失败时自动使用软件）

**关键配置**：
```kotlin
// ✅ 启用解码器回退
enableDecoderFallback = true

// ✅ 创建支持软件解码器的视频渲染器
MediaCodecVideoRenderer(
    context,
    mediaCodecSelector,
    allowedVideoJoiningTimeMs,
    enableDecoderFallback,  // ✅ 关键：允许回退到软件解码器
    ...
)
```

### 3. 优化 VideoPlayerManager

**当前实现**：
- ✅ 检测鸿蒙系统
- ✅ 自动重试机制（最多 2 次）
- ✅ 第二次重试时尝试软件解码器
- ✅ 详细的错误日志

**优化建议**：
- 在鸿蒙系统上，首次失败后立即尝试软件解码器
- 增加重试延迟（1500ms）给系统更多时间释放资源

### 4. Android Manifest 配置

**已配置**：
```xml
<!-- 视频播放硬件特性（可选） -->
<uses-feature android:name="android.hardware.video.output" android:required="false" />
<uses-feature android:name="android.software.video.decoder" android:required="false" />
```

**说明**：
- `required="false"` 表示不强制要求硬件解码器
- 允许应用在无硬件解码器的设备上运行（使用软件解码器）

## 使用说明

### 自动优化
应用会自动检测鸿蒙系统，并在需要时使用软件解码器。无需手动配置。

### 手动测试
如果需要手动测试软件解码器：

```dart
// 在 VideoPlayerManager 中
final controller = await VideoPlayerManager.instance.initVideoPlayer(
  url,
  forceSoftwareDecoder: true,  // ✅ 强制使用软件解码器
);
```

### 调试信息
查看日志以了解解码器选择：

```
🔍 [VideoPlayerManager] 设备检测: isHarmonyOS=true
⚠️ [VideoPlayerManager] 鸿蒙系统：尝试使用软件解码器
✅ [VideoPlayerManager] video_player 初始化成功
```

## 注意事项

### 1. 性能影响
- **软件解码器**：CPU 占用更高，但兼容性更好
- **硬件解码器**：性能更好，但可能在鸿蒙系统上不兼容

### 2. 视频格式
- 优先支持 H.264 编码
- H.265/HEVC 可能在部分设备上不支持

### 3. 分辨率限制
- 高分辨率视频（如 2560x1440）可能超出硬件解码能力
- 建议后端提供多分辨率版本

## 后续优化
