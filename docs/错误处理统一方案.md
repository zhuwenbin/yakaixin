# 错误处理统一方案

## 🎯 设计目标

**问题**：每个 Provider 都需要重复写类似的错误处理代码：
- `on DioException catch (e)`
- `on Exception catch (e)`
- `catch (e)`
- 使用 `ErrorMessageMapper.mapDioException(e)` 或 `ErrorMessageMapper.mapException(e)`

**解决方案**：创建统一的 `ErrorHandler` 类，Provider 层只需要一行代码即可处理所有错误。

---

## 📦 架构设计

### 错误处理流程

```
网络请求 → API拦截器 → DioException(error: "用户友好信息")
                              ↓
                    Provider catch (e)
                              ↓
                    ErrorHandler.handle(e)
                              ↓
                    返回用户友好错误信息
                              ↓
                    state = state.copyWith(error: errorMsg)
```

### 核心类

#### 1. `ErrorHandler` - 统一错误处理入口

**文件**: `lib/core/utils/error_handler.dart`

**职责**：
- 统一处理所有类型的错误（DioException、Exception、其他）
- 自动识别错误类型并调用对应的映射方法
- Provider 层无需关心错误类型

**API**：
```dart
// 统一处理错误，返回用户友好的错误信息
static String handle(Object error, {String defaultMessage = '操作失败，请稍后重试'})

// 处理错误并返回结果（用于需要返回值的场景）
static T handleAsResult<T>(Object error, T Function(String) onError, {String defaultMessage})
```

## ✅ 使用示例

### 示例 1: 标准 Provider 错误处理

**修改前**（❌ 冗余代码）：
```dart
Future<void> loadData() async {
  try {
    // 业务逻辑
  } on DioException catch (e) {
    // ✅ 优先处理 DioException（包含后端返回的业务错误信息）
    final errorMsg = ErrorMessageMapper.mapDioException(e);
    state = state.copyWith(error: errorMsg);
  } on Exception catch (e) {
    // ✅ Service层的Exception（已包含用户友好信息）
    final errorMsg = ErrorMessageMapper.mapException(e);
    state = state.copyWith(error: errorMsg);
  } catch (e) {
    // ✅ 其他未预期错误
    state = state.copyWith(error: '操作失败，请稍后重试');
  }
}
```

**修改后**（✅ 简洁代码）：
```dart
Future<void> loadData() async {
  try {
    // 业务逻辑
  } catch (e) {
    // ✅ 一行代码处理所有错误类型
    final errorMsg = ErrorHandler.handle(e);
    state = state.copyWith(error: errorMsg);
  }
}
```

### 示例 3: 返回 Result 的场景

**修改前**（❌ 冗余代码）：
```dart
Future<PaymentResult> createOrder() async {
  try {
    // 业务逻辑
    return PaymentResult.success(data);
  } on DioException catch (e) {
    final errorMsg = ErrorMessageMapper.mapDioException(e);
    return PaymentResult.error(errorMsg);
  } on Exception catch (e) {
    final errorMsg = ErrorMessageMapper.mapException(e);
    return PaymentResult.error(errorMsg);
  } catch (e) {
    return PaymentResult.error('操作失败，请稍后重试');
  }
}
```

**修改后**（✅ 简洁代码）：
```dart
Future<PaymentResult> createOrder() async {
  try {
    // 业务逻辑
    return PaymentResult.success(data);
  } catch (e) {
    // ✅ 使用 handleAsResult 简化代码
    return ErrorHandler.handleAsResult(
      e,
      (errorMsg) => PaymentResult.error(errorMsg),
    );
  }
}
```


### Step 2: 替换错误处理代码

**查找所有需要修改的地方**：

**替换规则**：
1. 删除 `on DioException catch (e)` 和 `on Exception catch (e)` 分支
2. 只保留 `catch (e)` 分支
3. 在 `catch (e)` 中使用 `ErrorHandler.handle(e)`

### 3. 易于扩展
- 新增错误类型只需要在 `ErrorHandler` 中添加处理逻辑
- Provider 层无需修改

### 4. 降低出错概率
- 减少重复代码，降低出错概率
- 统一的错误处理逻辑，确保一致性

---

## 📝 注意事项

1. **拦截器已处理 DioException**：
   - `ApiInterceptor` 已经将 DioException 转换为用户友好信息
   - `ErrorHandler` 直接使用 `error.error` 即可获取友好信息

2. **Service 层异常**：
   - Service 层抛出的 `Exception` 应该包含用户友好的错误信息
   - `ErrorHandler` 会直接使用这些信息

3. **兜底处理**：
   - 如果无法提取友好信息，`ErrorHandler` 会返回默认提示
   - 默认提示可以通过 `defaultMessage` 参数自定义
