# 登录后跳转问题分析

## 🔍 问题描述

**现象**：未登录状态在题库页面购买商品后，登录成功还是跳转到首页，而不是返回题库页面。

## 🐛 问题原因分析

### 根本原因

在 `app_router.dart` 的路由拦截器（`redirect`）中，有一个逻辑：

```dart
// 如果已登录且在登录页，跳转到主页
if (hasToken && isOnLoginPage) {
  return AppRoutes.mainTab;  // ❌ 这里强制跳转到首页
}
```

### 执行顺序问题

1. **用户在商品详情页点击购买**（未登录）
   - 跳转到登录页，传递 `returnPath: AppRoutes.mainTab`
   - `LoginCenterPage(returnPath: '/main-tab')` 被创建

2. **用户输入账号密码，点击登录**
   - `authProvider.loginWithSms()` 执行成功
   - Token 被保存到本地存储
   - `authProvider.isLoggedIn` 变为 `true`

3. **路由拦截器被触发**（在 `login_page.dart` 的 `Future.microtask` 之前）
   - 检测到 `hasToken == true` 且 `isOnLoginPage == true`
   - **强制返回 `AppRoutes.mainTab`**
   - 路由系统执行跳转到 `mainTab`

4. **`login_page.dart` 中的跳转逻辑执行**（但可能已经晚了）
   - `Future.microtask()` 中的 `context.go(returnPath)` 执行
   - 但此时路由拦截器已经强制跳转到 `mainTab` 了

### 问题根源

**路由拦截器的优先级高于页面内的跳转逻辑**，导致：
- 即使 `login_page.dart` 中设置了 `returnPath`
- 路由拦截器也会先执行，强制跳转到 `mainTab`
- 页面内的 `context.go(returnPath)` 可能被覆盖或无效

## ✅ 解决方案

### 方案1：在路由拦截器中检查 returnPath（推荐）

修改 `app_router.dart` 的 `redirect` 逻辑，如果检测到有 `returnPath`，优先使用 `returnPath`：

```dart
// 如果已登录且在登录页，检查是否有返回路径
if (hasToken && isOnLoginPage) {
  // ✅ 检查是否有返回路径（从路由 state 中获取）
  final extra = state.extra as Map<String, dynamic>?;
  final returnPath = extra?['returnPath'] as String?;
  
  if (returnPath != null && returnPath.isNotEmpty) {
    // ✅ 有返回路径，使用返回路径
    return returnPath;
  } else {
    // ✅ 没有返回路径，跳转到主页
    return AppRoutes.mainTab;
  }
}
```

### 方案2：延迟路由拦截器的执行

在 `login_page.dart` 中，使用 `WidgetsBinding.instance.addPostFrameCallback` 确保在路由拦截器之后执行：

```dart
// ✅ 延迟到下一帧执行，确保路由拦截器先执行完
WidgetsBinding.instance.addPostFrameCallback((_) {
  if (!mounted) return;
  
  if (returnPath != null && returnPath.isNotEmpty) {
    context.go(returnPath);
  } else {
    Navigator.of(context).pop();
  }
});
```

### 方案3：在登录成功后立即跳转（不等待路由拦截器）

在 `auth_provider.dart` 的登录方法中，登录成功后立即跳转：

```dart
// 在 loginWithSms 或 loginWithPhone 成功后
// 立即使用 ref.read(goRouterProvider).go(returnPath)
```

## 🎯 推荐方案

**推荐使用方案1**，因为：
1. 逻辑更清晰：路由拦截器统一处理路由跳转
2. 避免竞态条件：不会出现路由拦截器和页面跳转冲突
3. 易于维护：所有路由逻辑集中在一个地方

## 📝 修改步骤

1. 修改 `app_router.dart` 的 `redirect` 方法
2. 添加对 `returnPath` 的检查
3. 如果有 `returnPath`，使用 `returnPath`；否则使用 `mainTab`
4. 测试验证：从题库页面购买商品，登录后应该返回题库页面
