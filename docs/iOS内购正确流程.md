## 🎯 正确的 iOS 内购流程

### 完整时间线

```
步骤 1: 用户点击"确认支付"
    ↓
    UI: EasyLoading.show(status: '正在支付...')
    ↓
    
步骤 2: 应用发起购买请求
    ↓
    调用: _paymentService.pay()
    ↓
    内部调用: _iap.buyConsumable()
    ↓
    日志: "购买请求已发送，等待用户确认"
    ↓
    
步骤 3: StoreKit 弹出支付确认框（Apple 系统弹窗）
    ↓
    显示: 商品名称、价格、"购买"按钮
    ↓
    
步骤 4: 用户操作
    ↓
    输入密码 或 Face ID 验证
    ↓
    点击"购买"按钮（沙盒环境可能显示"完成支付"）
    ↓
    
步骤 5: Apple 处理支付
    ↓
    连接 App Store 服务器
    ↓
    验证账号
    ↓
    处理支付
    ↓
    扣款成功 ✅
    ↓
    Apple 提示: "您已支付完成"（沙盒环境）
    ↓
    
步骤 6: StoreKit 触发回调（关键！）
    ↓
    调用: _onPurchaseUpdate(purchaseDetails)
    ↓
    日志: "收到购买更新: 1 个"
    ↓
    日志: "购买状态：成功"
    ↓
    
步骤 7: 获取收据数据
    ↓
    receiptData = purchaseDetails.verificationData.serverVerificationData
    ↓
    日志: "票据服务数据: ..."
    ↓
    
步骤 8: 保存收据（安全备份）
    ↓
    IAPReceiptCache.saveFailedReceipt()
    ↓
    日志: "💾 验证前保存收据到缓存（安全备份）"
    ↓
    日志: "✅ 收据已安全备份"
    ↓
    
步骤 9: 后台验证收据
    ↓
    POST /c/pay/applepay
    ↓
    发送: receipt_data, flow_id, finance_body_id, product_id, transaction_id
    ↓
    日志: "开始后台验证..."
    ↓
    等待服务器响应...
    ↓
    
步骤 10: 验证成功
    ↓
    日志: "后台验证成功"
    ↓
    删除缓存: IAPReceiptCache.removeFailedReceipt()
    ↓
    日志: "🗑️ 验证成功，删除收据缓存"
    ↓
    
步骤 11: 完成交易
    ↓
    _iap.completePurchase(purchaseDetails)
    ↓
    日志: "内购流程完成"
    ↓
    
步骤 12: 通知 UI
    ↓
    callback(true, null)
    ↓
    completer.complete(PaymentResult.succeeded())
    ↓
    EasyLoading.dismiss()
    ↓
    跳转到支付成功页
```

## 🔍 当前问题分析

### 用户报告的现象

```
✅ 步骤 1-4: 正常
✅ 步骤 5: 输入密码，点击"完成支付"
✅ Apple 提示: "您已支付完成"
❌ 步骤 6: 回调丢失！
❌ UI 一直显示："正在验证..."
```

### 问题根源

**StoreKit 回调丢失**：
```
原因：
1. StoreKit 框架 Bug（沙盒环境偶尔会出现）
2. 支付成功但回调未触发
3. 应用无法知道支付结果
4. UI 一直等待回调
5. 没有超时机制（之前）
```

---

## 🔧 已修复的问题

### 修复 1: 删除不准确的 UI 提示

**之前的错误代码**：
```dart
// ❌ 错误：1.5秒后就显示"正在验证..."
Future.delayed(const Duration(milliseconds: 1500), () {
  EasyLoading.show(status: '正在验证...');
});
```

**问题**：
- 1.5秒时用户可能还在输入密码
- 根本还没开始验证
- 提示不准确

**修复后**：
```dart
// ✅ 正确：保持"正在支付..."直到收到回调
EasyLoading.show(status: '正在支付...');

// 等待回调
await completer.future;

// 收到回调后才关闭
EasyLoading.dismiss();
```

### 修复 2: 添加 30 秒超时保护

**之前的问题**：
```dart
// ❌ 没有超时机制
await completer.future;  // 永远等待
```

**修复后**：
```dart
// ✅ 添加 30 秒超时
final timeoutTimer = Timer(const Duration(seconds: 30), () {
  if (!completer.isCompleted) {
    isTimeout = true;
    print('\n⚠️ 支付超时（30秒无响应）');
    print('   如已完成支付，请重启应用查看权限');
    completer.complete(
      PaymentResult.failed('支付超时\n如已完成支付，请重启应用查看权限')
    );
  }
});

// 等待回调或超时
await completer.future;

// 如果正常收到回调，取消超时
timeoutTimer.cancel();
```

**效果**：
- 如果 30 秒内没有回调
- 自动结束等待
- 提示用户重启应用
- 重启后清理模式会处理
