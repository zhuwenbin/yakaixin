# 收据缓存策略优化

**优化时间**: 2025-01-25  
**优化目标**: 防止验证过程中应用中断导致收据丢失  
**严重程度**: 🟡 中危 → ✅ 已优化

---

## 🔍 问题分析

### 优化前的流程

```
支付成功
    ↓
开始验证 ← ⚠️ 风险窗口开始
    ↓
[验证过程中可能崩溃]
    ↓
验证失败 → 保存收据
    ↓
验证成功 → 完成交易
```

### 存在的风险 ⚠️

**时间线**：
```
1. 支付成功 (Line 325)
2. 调用验证方法 (Line 327)
3. 获取收据 (Line 373)
4. 开始验证 (Line 402) ← ⚠️ 收据还没保存
5. [应用崩溃/被杀死/网络超时]
6. ❌ 收据丢失
```

**风险窗口**：
- 从"开始验证"到"验证完成"之间
- 如果应用崩溃/被杀死
- 收据还没来得及保存
- 下次启动无法恢复
- **用户付款但权限未开通** ❌

### 具体场景

#### 场景 1：验证超时

```
用户购买 → 支付成功 → 开始验证 → 网络超时 30 秒
                                    ↓
                            用户等不及，关闭应用
                                    ↓
                            ❌ 收据丢失
```

#### 场景 2：应用崩溃

```
用户购买 → 支付成功 → 开始验证 → 应用崩溃
                                    ↓
                            ❌ 收据丢失
```

#### 场景 3：系统杀死应用

```
用户购买 → 支付成功 → 开始验证 → 系统内存不足，杀死应用
                                    ↓
                            ❌ 收据丢失
```

## ✅ 优化方案

### 新的策略：先保存，后验证

```
支付成功
    ↓
立即保存收据 ← ✅ 安全备份
    ↓
开始验证
    ↓
验证成功 → 删除缓存 ✅
    ↓
验证失败 → 保留缓存 ✅（已保存）
```

### 实现代码

```dart
Future<void> _verifyAndFinishTransaction(PurchaseDetails purchaseDetails) async {
  try {
    // 1. 获取收据数据
    receiptData = purchaseDetails.verificationData.serverVerificationData;
    
    // 2. ✅ 验证之前先保存收据（防止验证过程中应用崩溃）
    debugPrint('💾 验证前保存收据到缓存（安全备份）');
    await IAPReceiptCache.saveFailedReceipt(
      orderId: _currentOrderId!,
      goodsId: _currentGoodsId!,
      financeBodyId: _currentFinanceBodyId!,
      studentId: _currentStudentId!,
      receiptData: receiptData,
      productId: purchaseDetails.productID,
      transactionId: purchaseDetails.purchaseID ?? '',
    );
    debugPrint('✅ 收据已安全备份');
    
    // 3. 开始验证
    final verifyResult = await _verifyReceiptWithServer(...);
    
    if (!verifyResult['success']) {
      throw Exception('后台验证失败');
    }
    
    // 4. ✅ 验证成功，删除缓存
    debugPrint('🗑️ 验证成功，删除收据缓存');
    await IAPReceiptCache.removeFailedReceipt(_currentOrderId!);
    
    // 5. 完成交易
    await _iap.completePurchase(purchaseDetails);
    cb?.call(true, null);
    
  } catch (e) {
    // 6. ✅ 验证失败，收据已在验证前保存
    debugPrint('💾 收据已保存在缓存中，将在下次启动时自动重试');
    await _iap.completePurchase(purchaseDetails);
    cb?.call(false, '收据验证失败，已保存数据，将自动重试: $e');
  }
}
```

## ⚠️ 注意事项

### 1. 缓存清理时机

**验证成功时删除**：
```dart
// ✅ 验证成功，立即删除缓存
await IAPReceiptCache.removeFailedReceipt(_currentOrderId!);
```

**验证失败时保留**：
```dart
// ✅ 验证失败，保留缓存供下次重试
debugPrint('💾 收据已保存在缓存中，将在下次启动时自动重试');
```

### 2. 重复保存检查

`IAPReceiptCache.saveFailedReceipt` 已包含重复检查：

```dart
// 检查是否已存在相同订单
final exists = failed.any((item) {

  return data['orderId'] == orderId;
});

if (exists) {
  print('⚠️ 订单 $orderId 已存在于缓存中，跳过保存');
  return;
}
```

### 3. 磁盘空间

每个收据约 2-5KB，正常情况下不会占用太多空间。
