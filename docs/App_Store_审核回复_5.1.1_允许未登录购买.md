# App Store 审核回复 - Guideline 5.1.1

## 中文回复（建议使用）

### 方案 A：仅说明未登录可浏览（不主张账户关联）

尊敬的 App Review 团队：

感谢您的反馈，我们对此前版本未能完全符合审核要求表示歉意。

我们已完成修改，当前版本行为如下：

1. **允许用户未登录浏览商品数据**  
   用户无需注册或登录即可浏览应用内的商品信息、详情与价格等数据。

2. **可不登录继续浏览其他商品**  
   未登录用户可自由切换、继续浏览其他商品，不会被强制要求登录或注册。

3. **商品类型说明**  
   我们的商品为教育类内容；我们会在产品与审核要求范围内持续配合贵方指引。

如审核过程中需要进一步说明或测试步骤，我们随时配合提供。再次感谢您的耐心与指正。

## 备选：说明内购为账户关联型（若保持「购买前需登录」）

若产品设计上**必须**与用户账户绑定（试卷、课程等需账户才能做题/看课/同步进度），可在回复中增加以下说明，主张内购属于「账户关联型」，要求登录符合 5.1.1 中「注册与账户特定功能绑定」的例外：

**可增加段落（中文）：**

我们的应用内购买商品（如试卷、题库、课程等）均为**账户关联型**：内容与学习进度必须绑定到用户账户才能使用，无法在未绑定账户的情况下交付或使用。因此，我们在用户点击购买时要求登录，是为了将购买与账户绑定，符合 Guideline 5.1.1 中「需要分享个人信息的用户注册须为可选，或与账户特定功能绑定」的表述。我们已支持未登录用户自由浏览所有商品信息与详情，仅在进行购买及使用已购内容时要求登录。

**可增加段落（English）：**

Our in-app purchase products (e.g. exam papers, question banks, courses) are **account-based**: content and learning progress must be bound to a user account to be delivered and used; they cannot be provided or used without an account. We therefore require login when the user taps purchase so that the purchase is tied to the account, which aligns with Guideline 5.1.1 (“User registration that requires the sharing of personal information must be optional or tied to account-specific functionality”). We already allow unauthenticated users to browse all product information and details; login is only required when purchasing and when using purchased content.

**注意**：若审核仍认为应允许未登录完成支付，则需改为「未登录可完成购买，使用内容时再提示登录/注册」。
