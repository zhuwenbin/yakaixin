# App Store 5.1.1 购买流程合规方案

## 审核拒绝要点

- **Guideline 5.1.1**: 应用不能在允许用户访问**非账户专属**的内容和功能之前强制注册。
- **拒绝原因**: 应用仍要求用户在购买**非账户绑定**的内购产品前，使用个人信息注册。

## 合规目标

- 用户**不必先注册/登录**即可完成「购买」行为（即：允许先完成支付，再可选登录绑定）。
- 可在合适位置说明「登录后可在多设备使用」，并随时提供「去登录」入口，但不强制在购买前登录。

## 方案可行性

### 用户方案

「加一个详情页，在详情页提示登录，并下订单购买」：

- **可行点**：把「点击即弹登录」改为「先进入购买详情/确认页，在该页提示登录并支持下订单」，符合「在详情页提示登录并下订单」的表述。
- **注意**：若下订单**仍然必须**先登录（后端 createOrder 必须 studentId），则仅加一层详情页无法从根本上满足 5.1.1；审核仍可能认为「购买前强制注册」。要彻底合规，需满足其一：
  1. **后端支持未登录下单**：例如用设备 ID / 匿名 ID 创建订单，或订单创建时 studentId 可选，支付成功后再绑定账号；
  2. **或使用 Apple IAP**：用户仅用 Apple ID 完成支付，应用内不要求「注册」；支付成功后用收据到后端兑换，后端支持仅凭收据发放权益（不强制传 studentId）。

### 当前实现

- **前端**：增加「购买确认页」；从题库/商品详情点击「购买」或「开始冲刺做题」时，**不再先弹登录框**，改为进入购买确认页；在该页展示商品与价格、文案「登录后可在多设备使用，您可随时登录绑定」、以及「去登录」「立即购买」；点击「立即购买」再走统一支付流程。
- **支付流程**：`PaymentFlowManager` 与 `createOrder` 当前仍依赖 `studentId`（后端必填）。未登录时若直接调起下单会失败，因此会在「点击立即购买」时再提示登录。若要实现**完全不强制登录即可完成购买**，需后端配合（见上）。

## 风险点与修改清单

| 位置 | 风险 | 处理 |
|------|------|------|
| 历年真题详情 `secret_real_detail_page` | 点击「开始冲刺做题」未购买时先弹登录 | 改为进入购买确认页，不再先弹登录 |
| 商品详情 `goods_detail_page` | 点击购买前检查登录 | 未登录时进入购买确认页 |
| 课程详情 `course_goods_detail_page` | 同上 | 同上 |
| 科目模考详情 `subject_mock_detail_page` | 同上 | 同上 |
| 题库页 `question_bank_page` | 打卡/刷题前检查登录 | 保持：打卡、刷题为账户相关功能，可要求登录 |
| 支付流程 `payment_flow_manager` | 无 userInfo 即弹登录并 return | 保持；若后端支持未登录下单，再改为传 deviceId/空 studentId |
| 模拟考场 `simulated_exam_room_page` | 未购买时跳转 courseGoodsDetail，无直接购买前登录弹窗 | 若从该页进入的课程详情有购买确认页即可 |

## 已做修改摘要

1. **新增购买确认页**（`lib/features/order/views/purchase_confirm_page.dart`）：展示商品、价格、文案「登录后可在多设备使用，您可随时登录绑定账户」、链接「去登录」、按钮「立即购买」；未登录可访问（路由已加入公开页）。
2. **历年真题**（`secret_real_detail_page.dart`）：未购买时点击「开始冲刺做题」→ 进入购买确认页（不先弹登录）；已购买点击做题仍校验登录（学习为账户相关）。页内增加非阻断式「登录后可在多设备使用」+「去登录」。
3. **商品详情**（`goods_detail_page.dart`）：未登录点击购买 → 进入购买确认页；已登录仍直接下单。
4. **课程详情**（`course_goods_detail_page.dart`）：未登录点击报名/购买 → 进入购买确认页。
5. **科目模考详情**（`subject_mock_detail_page.dart`）：未登录点击购买 → 进入购买确认页；开始练习仍校验登录。
6. **路由**：新增 `AppRoutes.purchaseConfirm`，并在 redirect 中列为公开页，未登录可进入。

## 其他 5.1.1 风险点（已排查）

| 位置 | 行为 | 结论 |
|------|------|------|
| 题库页 `question_bank_page` | 打卡、刷题前检查登录 | 合理：打卡/刷题为账户相关，可要求登录 |
| `PaymentFlowManager` | 创建订单前要求 userInfo/studentId | 当前后端必填，保持；若后端支持未登录下单再改 |
| 模拟考场 `simulated_exam_room_page` | 未购买时跳转课程详情 | 课程详情已改为先进购买确认页，无额外风险 |
| 首页/其他入口 | 进入商品详情均可不登录浏览 | 已允许 |

完全满足 5.1.1 的后续可选工作：后端支持未登录下单或 Apple IAP 先付后绑定，前端再移除支付流程中的「必须登录」校验。
