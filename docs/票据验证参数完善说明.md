## 🎯 解决的问题

### 问题描述
票据（Receipt）包含所有历史购买记录，后端需要明确知道：
1. **当前验证的是哪个产品** (`product_id`)
2. **当前验证的是哪次交易** (`transaction_id`)

### 解决方案
在发送票据验证请求时，附带以下参数：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `receipt_data` | Apple票据（Base64） | `MIIUVQYJKoZ...` |
| `flow_id` | 流水ID（订单ID） | `123456789` |
| `finance_body_id` | 财务主体ID | `789` |
| **`product_id`** | **当前购买的产品ID** | `com.yakaixin.yakaixin.595918548996463432` |
| **`transaction_id`** | **当前交易ID** | `2000001080123456` |

## 📱 后端如何使用这些参数

### 验证流程

```python
def verify_receipt(receipt_data, flow_id, product_id, transaction_id):
    # 1. 发送票据到Apple验证服务器
    apple_response = send_to_apple(receipt_data)
    
    # 2. Apple返回所有历史内购记录
    all_purchases = apple_response['receipt']['in_app']
    # 例如：
    # [
    #   {'product_id': '...463431', 'transaction_id': '2000001079...'},
    #   {'product_id': '...463432', 'transaction_id': '2000001080...'},
    # ]
    
    # 3. ✅ 根据 transaction_id 找到当前交易
    current_purchase = find_by_transaction_id(all_purchases, transaction_id)
    
    # 4. ✅ 验证 product_id 是否匹配
    if current_purchase['product_id'] != product_id:
        return error("产品ID不匹配")
    
    # 5. ✅ 验证通过，开通权限
    grant_permission(flow_id, product_id)
    return success()
```

## 🔍 为什么需要这两个参数？

### 用户遇到的问题

> "为什么票据验证格式中有之前的 `com.yakaixin.yakaixin.595918548996463431` 的商品，而不是现在的 `com.yakaixin.yakaixin.595918548996463432` 商品？"

**原因**：Apple的票据（Receipt）中包含**所有历史购买记录**（In-App Purchases），而不仅仅是当前这笔购买。

### Apple Receipt 的结构

```json
{
  "receipt": {
    "bundle_id": "com.yakaixin.yakaixin",
    "in_app": [
      {
        "product_id": "com.yakaixin.yakaixin.595918548996463431",
        "transaction_id": "2000001079476252",
        "purchase_date": "2025-11-20"
      },
      {
        "product_id": "com.yakaixin.yakaixin.595918548996463432",
        "transaction_id": "2000001080123456",
        "purchase_date": "2025-12-12"
      }
    ]
  }
}
```

### 之前的问题

在添加 `product_id` 和 `transaction_id` 之前：
- ❌ 后端不知道当前验证的是哪一笔购买
- ❌ 可能将旧的购买记录（463431）误认为是当前购买
- ❌ 无法准确关联订单流水

### 现在的解决方案

添加参数后：
- ✅ 通过 `transaction_id` 精确定位当前交易
- ✅ 通过 `product_id` 验证产品匹配（确保是 463432 而非 463431）
- ✅ 避免重复处理历史购买
- ✅ 准确关联订单流水
