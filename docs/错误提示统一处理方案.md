# 错误提示统一处理方案（完整版）

## 🎯 目标

解决用户经常看到技术性错误提示的问题，确保：
- ❌ 不显示 `DioException [connection error]: xxx`
- ❌ 不显示 `type 'String' is not a subtype of type 'int'`
- ❌ 不显示 `bad response`
- ✅ 只显示用户可理解的友好提示：如"网络连接失败"、"账号密码不匹配"等

---

## 🛠️ 核心组件

### 1. API拦截器 (`lib/core/network/api_interceptor.dart`)

#### 职责
- 拦截所有HTTP响应和错误
- 提取后端返回的业务错误信息（`msg`字段）
- 将技术性网络错误转换为用户友好提示

#### 关键方法

**`onResponse()` - 处理业务错误**

```dart
@override
void onResponse(Response response, ResponseInterceptorHandler handler) {
  if (response.data is Map) {
    final data = response.data as Map<String, dynamic>;
    
    if (data.containsKey('code')) {
      final code = data['code'];
      
      // ✅ 业务失败：code != 100000
      if (code != 100000 && code != 0) {
        final msg = data['msg'];
        final errorMessage = _extractErrorMessage(msg);  // ← 提取后端msg
        
        handler.reject(
          DioException(
            requestOptions: response.requestOptions,
            error: errorMessage,  // ← 后端错误信息，用户可理解
            type: DioExceptionType.badResponse,
            response: response,
          ),
        );
        return;
      }
    }
  }
  super.onResponse(response, handler);
}

/// 提取错误消息
/// msg可能是字符串或数组
String _extractErrorMessage(dynamic msg) {
  if (msg == null) return '哎呦,出错啦~~';
  
  if (msg is String) {
    return msg.isEmpty ? '哎呦,出错啦~~' : msg;
  }
  
  if (msg is List) {
    if (msg.isEmpty) return '哎呦,出错啦~~';
    return msg.join(', ');  // ← 数组格式的错误信息用逗号连接
  }
  
  return '哎呦,出错啦~~';
}
```

**`onError()` - 处理网络错误**

```dart
@override
void onError(DioException err, ErrorInterceptorHandler handler) {
  // ✅ 使用 ErrorMessageMapper 将错误转换为用户友好提示
  final userFriendlyMessage = ErrorMessageMapper.mapDioException(err);
  
  // 创建新的DioException，包含用户友好的错误信息
  final newError = DioException(
    requestOptions: err.requestOptions,
    error: userFriendlyMessage,  // ← 用户友好提示
    type: err.type,
    response: err.response,
  );

  super.onError(newError, handler);
}
```

### 2. 错误消息映射器 (`lib/core/utils/error_message_mapper.dart`)

#### 职责
- 将技术性错误转换为用户友好的提示
- 过滤掉不应该展示给用户的技术细节

#### 核心方法

```dart
/// 将 DioException 转换为用户友好的错误提示
static String mapDioException(DioException error) {
  // 优先使用拦截器已处理的错误信息
  if (error.error != null && error.error is String) {
    final errorMsg = error.error as String;
    // ✅ 过滤掉技术性错误
    if (!_isTechnicalError(errorMsg)) {
      return errorMsg;
    }
  }

  // 根据错误类型返回友好提示
  switch (error.type) {
    case DioExceptionType.connectionTimeout:
      return '连接超时，请检查网络后重试';
    
    case DioExceptionType.connectionError:
      return '网络连接失败，请检查网络设置';
    
    case DioExceptionType.badResponse:
      return _mapBadResponse(error);
    
    // ... 更多错误类型映射
  }
}

/// 判断是否为技术性错误（需要转换为友好提示）
static bool _isTechnicalError(String message) {
  final technicalKeywords = [
    'bad response',
    'type \'',
    'is not a subtype',
    'FormatException',
    'SocketException',
    'Failed host lookup',
    // ... 更多技术关键词
  ];
  
  return technicalKeywords.any((keyword) => message.contains(keyword));
}

/// 处理 badResponse 错误
static String _mapBadResponse(DioException error) {
  final statusCode = error.response?.statusCode;
  
  switch (statusCode) {
    case 400:
      return '请求参数错误';
    case 401:
      return '登录已失效，请重新登录';
    case 403:
      return '没有访问权限';
    case 404:
      return '请求的内容不存在';
    case 500:
      return '服务器错误，请稍后重试';
    // ... 更多状态码
    default:
      // ✅ 尝试从响应中提取后端的错误信息
      if (error.response?.data is Map) {
        final data = error.response!.data as Map<String, dynamic>;
        final msg = data['msg'] ?? data['message'];
        if (msg != null) {
          if (msg is String && msg.isNotEmpty) {
            return msg;
          }
          if (msg is List && msg.isNotEmpty) {
            return msg.join(', ');
          }
        }
      }
      return '服务器错误($statusCode)，请稍后重试';
  }
}
```

### 3. Provider层标准处理模式

#### ✅ 标准错误捕获代码

```dart
Future<void> loadData() async {
  try {
    state = state.copyWith(isLoading: true, error: null);
    
    // 调用 Service
    final result = await _service.getData();
    
    state = state.copyWith(
      data: result,
      isLoading: false,
    );
  } on DioException catch (e) {
    // ✅ DioException: 使用拦截器已处理的友好信息
    final errorMsg = e.error?.toString() ?? '操作失败，请稍后重试';
    state = state.copyWith(
      isLoading: false,
      error: errorMsg,
    );
  } catch (e) {
    // ✅ 其他异常: 使用兜底提示
    state = state.copyWith(
      isLoading: false,
      error: '操作失败，请稍后重试',
    );
  }
}
```

#### ❌ 错误示例

```dart
// ❌ 错误1：直接使用 e.toString()
} catch (e) {
  state = state.copyWith(error: e.toString());  // ← 会显示技术错误
}

// ❌ 错误2：不区分 DioException 和其他异常
} catch (e) {
  final errorMsg = e.toString();  // ← 无法获取拦截器处理的友好信息
  state = state.copyWith(error: errorMsg);
}

// ❌ 错误3：直接使用 e.message
} on DioException catch (e) {
  state = state.copyWith(error: e.message);  // ← e.message 可能为 null
}
```

### 4. View层错误显示

#### 方式1：使用 EasyLoading/Toast

```dart
// ✅ 在 Provider 中调用 Service 后直接显示提示
Future<void> submitData() async {
  try {
    EasyLoading.show(status: '提交中...');
    
    await _service.submit();
    
    EasyLoading.dismiss();
    EasyLoading.showSuccess('提交成功');
  } on DioException catch (e) {
    EasyLoading.dismiss();
    final errorMsg = e.error?.toString() ?? '提交失败';
    EasyLoading.showError(errorMsg);  // ← 显示友好提示
    rethrow;
  } catch (e) {
    EasyLoading.dismiss();
    EasyLoading.showError('提交失败，请稍后重试');
    rethrow;
  }
}
```

#### 方式2：使用 state.error + ref.listen

```dart
// Provider
Future<void> loadData() async {
  try {
    state = state.copyWith(isLoading: true, error: null);
    final result = await _service.getData();
    state = state.copyWith(data: result, isLoading: false);
  } on DioException catch (e) {
    final errorMsg = e.error?.toString() ?? '加载失败';
    state = state.copyWith(isLoading: false, error: errorMsg);
  }
}

// View
class MyPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final myState = ref.watch(myProvider);
    
    // ✅ 监听错误状态并显示提示
    ref.listen<MyState>(
      myProvider,
      (previous, next) {
        if (next.error != null) {
          EasyLoading.showError(next.error!);  // ← 显示友好提示
        }
      },
    );
    
    return Scaffold(...);
  }
}
```

## 📊 错误处理优先级

### 优先级顺序（从高到低）

1. **后端业务错误消息** (最高优先级)
   - 来源：`response.data['msg']`
   - 示例：`"手机号格式错误"`, `"验证码已过期"`, `"账号密码不匹配"`
   - ✅ 直接显示给用户（这是后端明确返回的用户可读信息）

2. **HTTP状态码映射**
   - 401 → `"登录已失效，请重新登录"`
   - 404 → `"请求的内容不存在"`
   - 500 → `"服务器错误，请稍后重试"`

3. **网络错误映射**
   - 连接超时 → `"连接超时，请检查网络后重试"`
   - 网络断开 → `"网络连接失败，请检查网络设置"`

4. **兜底提示** (最低优先级)
   - 无法识别的错误 → `"哎呦，出错啦~~"`

## 🔧 常见问题排查

### 问题1: 仍然显示 "DioException [connection error]"

**原因**: Provider 中使用了 `e.toString()` 而不是 `e.error?.toString()`

**解决**:
```dart
// ❌ 错误
} on DioException catch (e) {
  state = state.copyWith(error: e.toString());
}

// ✅ 正确
} on DioException catch (e) {
  final errorMsg = e.error?.toString() ?? '操作失败';
  state = state.copyWith(error: errorMsg);
}
```

---

### 问题2: 后端返回的错误信息没有显示

**原因**: 拦截器中 `_extractErrorMessage()` 逻辑有问题

**检查**:
```dart
// 确保 _extractErrorMessage 正确处理 msg
String _extractErrorMessage(dynamic msg) {
  if (msg == null) return '哎呦,出错啦~~';
  
  if (msg is String) {
    return msg.isEmpty ? '哎呦,出错啦~~' : msg;
  }
  
  if (msg is List) {
    if (msg.isEmpty) return '哎呦,出错啦~~';
    return msg.join(', ');  // ✅ 数组拼接
  }
  
  return '哎呦,出错啦~~';
}
```

---

### 问题3: 错误提示显示了两次

**原因**: Provider 和 View 都显示了错误提示

**解决**: 选择一种方式
- 要么在 Provider 中使用 `EasyLoading.showError()`
- 要么在 View 中使用 `ref.listen()` 监听 `state.error`

**不要同时使用两种方式！**

## 📝 总结

### 核心原则

1. **拦截器统一处理**
   - 在拦截器中将所有技术错误转换为友好提示
   - Provider 只需要使用 `e.error?.toString()`

2. **后端错误优先**
   - 后端返回的 `msg` 字段直接展示（这是用户可理解的）
   - 技术错误需要映射转换

3. **分层职责明确**
   - **拦截器**: 转换错误信息
   - **Provider**: 捕获异常并更新状态
   - **View**: 显示友好提示

4. **兜底机制**
   - 所有 Provider 必须有 `catch (e)` 兜底
   - 所有错误提示都有默认值（避免显示 null）
