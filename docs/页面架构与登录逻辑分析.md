# 页面架构与登录逻辑分析

## 📱 App 页面架构

### 1. 路由架构

使用 **GoRouter** 进行路由管理，采用声明式路由配置。

```
应用启动
  ↓
SplashPage (启动页)
  ↓
MainTabPage (主框架 - TabBar)
  ├── HomePage (首页-刷题)
  ├── QuestionBankPage (题库首页)
  ├── CoursePage (课程首页)
  └── ProfilePage (我的)
```

### 2. 主框架结构

**MainTabPage** (`/main-tab`) 是应用的主框架，包含：
- **IndexedStack**：保持 4 个 Tab 页面的状态
- **BottomNavigationBar**：底部导航栏
- 4 个 Tab 页面：
  1. `HomePage` - 首页（刷题）
  2. `QuestionBankPage` - 题库首页
  3. `CoursePage` - 课程首页
  4. `ProfilePage` - 我的

### 3. 路由拦截机制

**app_router.dart** 中的 `redirect` 函数负责：
- ✅ **未登录拦截**：未登录用户访问需要登录的页面时，跳转到登录页
- ✅ **公开页面**：允许未登录用户访问的页面（如商品详情、首页等）
- ⚠️ **已登录处理**：已登录用户在登录页时，不立即跳转，让 `login_page.dart` 处理

### 4. 页面类型分类

#### 公开页面（无需登录）
- `AppRoutes.splash` - 启动页
- `AppRoutes.mainTab` - 主框架（TabBar）
- `AppRoutes.home` - 首页
- `AppRoutes.goodsDetail` - 商品详情
- `AppRoutes.courseGoodsDetail` - 课程详情
- `AppRoutes.secretRealDetail` - 真题详情
- `AppRoutes.subjectMockDetail` - 科目详情
- `AppRoutes.simulatedExamRoom` - 模考详情
- 其他设置、协议页面

#### 需要登录的页面
- 订单相关页面
- 个人中心部分功能
- 购买/支付流程

## 🔐 登录逻辑流程

### 1. 登录触发场景

当用户执行以下操作时，如果未登录，会弹出登录提示：
- 点击"购买"按钮
- 点击"开始练习"按钮
- 点击"打卡"按钮
- 点击"每日一测"按钮
- 点击"章节练习"按钮
- 其他需要登录的操作

### 2. 登录流程

```
用户点击需要登录的操作
  ↓
检查登录状态 (ref.read(authProvider).isLoggedIn)
  ↓
未登录 → 显示登录对话框 (ConfirmDialog)
  ↓
用户点击"去登录"
  ↓
获取当前页面路径 (currentPath)
  ↓
判断是否有上一页 (Navigator.canPop())
  ↓
设置返回路径 (returnPath)
  ↓
跳转到登录页 (context.push(AppRoutes.loginCenter, extra: {'returnPath': returnPath}))
  ↓
登录页保存 returnPath 到 Provider
  ↓
用户输入账号密码并登录
  ↓
登录成功 (authProvider.isLoggedIn = true)
  ↓
获取 returnPath (从 widget.returnPath 或 Provider)
  ↓
跳转到 returnPath (context.go(returnPath))
```

### 3. 返回路径传递机制

#### 方式 1: 通过 `extra` 参数传递
```dart
context.push(
  AppRoutes.loginCenter,
  extra: {'returnPath': returnPath},
);
```

#### 方式 2: 通过 `loginReturnPathProvider` 全局状态
```dart
// 保存到 Provider
ref.read(loginReturnPathProvider.notifier).state = returnPath;

// 从 Provider 读取
final returnPath = ref.read(loginReturnPathProvider);
```

#### 方式 3: 通过 `LoginPage` 的 `returnPath` 参数
```dart
LoginPage(returnPath: returnPath)
```

**优先级**：`widget.returnPath` > `ModalRoute.arguments` > `loginReturnPathProvider`

## ❌ 问题分析：购买页面登录后跳转到首页

### 问题描述

在商品详情页（`/goods-detail`）点击"购买"按钮，未登录时弹出登录提示，登录成功后**跳转到了首页**（`/main-tab`），而不是返回到商品详情页。

### 问题根源

**位置**：`goods_detail_page.dart` 的 `_showLoginDialog()` 方法

**错误代码**：
```dart
void _showLoginDialog() {
  ConfirmDialog.show(
    context,
    onConfirm: () {
      final router = GoRouter.of(context);
      final currentPath = router.routerDelegate.currentConfiguration.uri.path;
      
      // ❌ 问题在这里：如果 canPop 为 true，就返回 mainTab
      final canPop = Navigator.of(context).canPop();
      final returnPath = canPop 
          ? AppRoutes.mainTab  // ❌ 错误：应该返回当前商品详情页路径
          : currentPath;
      
      context.push(
        AppRoutes.loginCenter,
        extra: {'returnPath': returnPath},
      );
    },
  );
}
```

### 问题分析

1. **错误假设**：代码假设如果有上一页（`canPop = true`），就应该返回 `mainTab`（题库页面）
2. **实际情况**：
   - 用户从题库页面（`mainTab`）跳转到商品详情页（`/goods-detail`）
   - `canPop = true`（因为有上一页）
   - `returnPath` 被错误地设置为 `AppRoutes.mainTab`
   - 登录成功后，跳转到 `mainTab`，而不是商品详情页

3. **正确的逻辑应该是**：
   - 无论是否有上一页，都应该返回**当前商品详情页的完整路径**（包括参数）
   - 商品详情页的路径应该是：`/goods-detail?goodsId=xxx&type=xxx`

### 解决方案

#### 方案 1: 使用当前完整路径（推荐）

```dart
void _showLoginDialog() {
  ConfirmDialog.show(
    context,
    onConfirm: () {
      final router = GoRouter.of(context);
      // ✅ 获取完整的当前路径（包括查询参数）
      final currentLocation = router.routerDelegate.currentConfiguration.uri.toString();
      
      // ✅ 或者使用 GoRouter 的 location 方法
      final returnPath = router.routerDelegate.currentConfiguration.uri.path;
      
      // ✅ 如果有查询参数，需要包含
      final queryParams = router.routerDelegate.currentConfiguration.uri.queryParameters;
      String fullPath = returnPath;
      if (queryParams.isNotEmpty) {
        final queryString = queryParams.entries
            .map((e) => '${e.key}=${e.value}')
            .join('&');
        fullPath = '$returnPath?$queryString';
      }
      
      print('🔄 [商品详情登录] 返回路径: $fullPath');
      
      context.push(
        AppRoutes.loginCenter,
        extra: {'returnPath': fullPath},
      );
    },
  );
}
```

#### 方案 2: 使用 GoRouter 的 `go()` 方法保存完整状态

```dart
void _showLoginDialog() {
  ConfirmDialog.show(
    context,
    onConfirm: () {
      // ✅ 保存当前路由的完整信息（包括参数）
      final router = GoRouter.of(context);
      final currentState = router.routerDelegate.currentConfiguration;
      
      // ✅ 构建完整的返回路径
      final returnPath = currentState.uri.toString();
      
      print('🔄 [商品详情登录] 返回路径: $returnPath');
      
      context.push(
        AppRoutes.loginCenter,
        extra: {'returnPath': returnPath},
      );
    },
  );
}
```

#### 方案 3: 传递商品参数，登录后重新构建路径

```dart
void _showLoginDialog() {
  ConfirmDialog.show(
    context,
    onConfirm: () {
      // ✅ 获取商品详情页的参数
      final goodsId = widget.goodsId ?? widget.productId;
      final type = widget.active; // 或其他类型参数
      
      // ✅ 构建返回路径（包含参数）
      String returnPath = AppRoutes.goodsDetail;
      if (goodsId != null) {
        returnPath = '$returnPath?goodsId=$goodsId';
        if (type != null) {
          returnPath = '$returnPath&type=$type';
        }
      }
      
      print('🔄 [商品详情登录] 返回路径: $returnPath');
      
      context.push(
        AppRoutes.loginCenter,
        extra: {'returnPath': returnPath},
      );
    },
  );
}
```

## 🔧 修复建议

### 1. 统一登录对话框逻辑

创建一个统一的登录对话框工具方法：

```dart
// lib/core/utils/login_helper.dart
class LoginHelper {
  /// 显示登录对话框并跳转到登录页
  /// [returnPath] 登录成功后的返回路径（可选，默认使用当前路径）
  static void showLoginDialog(
    BuildContext context, {
    String? returnPath,
  }) {
    ConfirmDialog.show(
      context,
      title: '需要登录',
      content: '此操作需要登录账户，请先登录后再试',
      confirmText: '去登录',
      cancelText: '取消',
      onConfirm: () {
        // ✅ 如果没有指定 returnPath，使用当前完整路径
        final router = GoRouter.of(context);
        final finalReturnPath = returnPath ?? 
            router.routerDelegate.currentConfiguration.uri.toString();
        
        print('🔄 [登录对话框] 返回路径: $finalReturnPath');
        
        context.push(
          AppRoutes.loginCenter,
          extra: {'returnPath': finalReturnPath},
        );
      },
    );
  }
}
```

### 2. 修复所有商品详情页

在所有商品详情页中使用统一的登录对话框：

```dart
// 替换所有 _showLoginDialog() 方法
void _showLoginDialog() {
  LoginHelper.showLoginDialog(context);
}
```

### 3. 确保登录后正确跳转

在 `login_page.dart` 中，确保使用完整的路径（包括参数）进行跳转：

```dart
// ✅ 登录成功后跳转
if (returnPath != null && returnPath.isNotEmpty) {
  // ✅ 使用 context.go() 跳转到完整路径（包括参数）
  context.go(returnPath);
}
```

## 📊 总结

### 问题根源
- ❌ 错误地假设有上一页就应该返回 `mainTab`
- ❌ 没有保存商品详情页的完整路径（包括参数）
- ❌ 登录成功后跳转的路径不完整

### 解决方案
- ✅ 使用当前页面的完整路径（包括查询参数）作为 `returnPath`
- ✅ 统一登录对话框逻辑，避免重复代码
- ✅ 确保登录成功后使用完整路径跳转
