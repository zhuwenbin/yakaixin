# 🧹 清理模式票据保存说明

## 问题背景

在 iOS 内购流程中，当应用初始化或购买前，会调用清理未完成的交易（`_clearPendingTransactions()`）。在清理模式下，系统会直接完成待处理的交易而不进行验证。

用户希望在清理模式下也能保存票据数据到文件，以便调试和追踪。

---

## 解决方案

### 修改位置

**文件**: `lib/features/payment/services/iap_service.dart`

**修改内容**:

1. **方法签名修改**（第328行）：
   ```dart
   // 修改前
   void _onPurchaseUpdate(List<PurchaseDetails> purchaseDetailsList) {
   
   // 修改后
   Future<void> _onPurchaseUpdate(List<PurchaseDetails> purchaseDetailsList) async {
   ```

2. **清理模式逻辑增强**（第348-367行）：
   ```dart
   // 🧹 清理模式：直接完成交易，不进行验证
   if (_isCleaningMode) {
     print('   🧹 清理模式：直接完成此交易');
     
     // 💾 保存票据数据到文件（用于调试）
     try {
       print('   📱 尝试获取并保存票据数据...');
       final receiptData = await _getReceiptData();
       if (receiptData != null && receiptData.isNotEmpty) {
         await _saveReceiptToFile(receiptData);
         print('   ✅ 票据已保存到文件');
       } else {
         print('   ⚠️ 票据数据为空，跳过保存');
       }
     } catch (e) {
       print('   ⚠️ 保存票据失败（不影响清理）: $e');
     }
     
     _iap.completePurchase(purchaseDetails);
     print('   ✅ 交易已完成（跳过验证）');
     
     // 🏁 最后一个交易处理完，通知清理完成
     if (i == purchaseDetailsList.length - 1 && _cleaningCompleter != null && !_cleaningCompleter!.isCompleted) {
       print('   🏁 所有交易处理完毕，通知清理完成');
       _cleaningCompleter!.complete();
     }
     continue; // 跳过下面的验证流程
   }
   ```

## 注意事项

### ⚠️ 清理模式特点

1. **不验证票据**
   - 只完成交易，不调用后端验证
   - 不会开通用户权限
   - 避免重复购买错误

2. **不缓存到 SharedPreferences**
   - 清理模式的票据不会保存到补发缓存
   - 仅保存到 `piaoju.txt` 文件用于调试
   - 不会触发自动补发重试

3. **票据可能为空**
   - 如果 App Store 收据不存在
   - 日志会显示：`⚠️ 票据数据为空，跳过保存`
   - 这是正常情况，不影响清理流程

### ✅ 安全措施

1. **错误不影响清理**
   ```dart
   try {
     // 保存票据
   } catch (e) {
     print('⚠️ 保存票据失败（不影响清理）: $e');
   }
   // 无论成功失败，都继续完成交易
   _iap.completePurchase(purchaseDetails);
   ```

2. **异步处理**
   - `_onPurchaseUpdate` 改为 `async` 方法
   - 使用 `await` 等待票据保存完成
   - 确保保存完成后再完成交易
