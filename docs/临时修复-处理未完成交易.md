# 临时修复：自动处理未完成交易

**修复时间**: 2025-01-25  
**问题**: `storekit_duplicate_product_object` 错误  
**解决方案**: 检测到错误时主动处理 pending transaction

---

## 🔴 问题描述

### 错误信息

```
购买失败: PlatformException(storekit_duplicate_product_object, 
There is a pending transaction for the same product identifier. 
Please either wait for it to be finished or finish it manually using `completePurchase`...)

错误: 该商品已购买，无需重复购买
```

### 问题原因

1. 之前的购买成功但未完成验证
2. 交易处于 pending 状态
3. StoreKit 阻止重复购买相同产品

---

## ✅ 临时修复方案

### 修复策略

**检测到错误时，主动触发恢复购买来处理 pending transaction**

```dart
Future<void> _handlePendingTransaction(double amount, String studentId) async {
  try {
    debugPrint('🔄 开始处理 pending transaction...');
    
    // 1. 触发恢复购买
    //    - 会触发 _onPurchaseUpdate 回调
    //    - 回调中会验证收据并完成交易
    await _iap.restorePurchases();
    
    // 2. 等待处理完成（给足够时间验证）
    await Future.delayed(const Duration(seconds: 3));

  } catch (e) {
    cb?.call(false, '检测到未完成的交易，请重启应用');
  }
}
```

## 🎯 工作原理

### 1. 触发恢复购买

```dart
await _iap.restorePurchases();
```

**作用**：
- 触发 StoreKit 检查所有 pending transaction
- 对每个 pending transaction 触发 `_onPurchaseUpdate` 回调

### 2. 回调处理

```dart
Future<void> _onPurchaseUpdate(List<PurchaseDetails> purchaseDetailsList) async {
  for (final purchaseDetails in purchaseDetailsList) {
    switch (purchaseDetails.status) {
      case PurchaseStatus.purchased:
        // ✅ 验证收据并完成交易
        _verifyAndFinishTransaction(purchaseDetails);
        break;
    }
  }
}
```

**流程**：
- 检测到 `PurchaseStatus.purchased`
- 保存收据（优化后的策略）
- 验证收据
- 验证成功 → 删除缓存 → 完成交易
- 验证失败 → 保留缓存 → 完成交易

### 3. 等待完成

```dart
await Future.delayed(const Duration(seconds: 3));
```

**原因**：
- 验证过程需要时间（网络请求）
- 给足够时间让验证流程完成
- 3秒通常足够完成验证

## ⚠️ 注意事项

### 1. 为什么是"临时修复"？

**原因**：
- 理想情况下不应该出现 pending transaction
- 优化后的策略（先保存收据）应该避免这个问题
- 但对于已经存在的 pending transaction，需要这个修复

**长期方案**：
- ✅ 验证前先保存收据（已实现）
- ✅ 清理模式保存收据（已实现）
- ✅ 应用启动时自动补发（已实现）

### 2. 为什么等待 3 秒？

**原因**：
- 验证过程需要网络请求
- 后台验证通常 1-2 秒
- 3 秒是一个安全的等待时间

**可能的情况**：
- ✅ 验证成功：权限已开通
- ❌ 验证失败：收据已保存，下次启动补发
- ⏳ 验证超时：收据已保存，下次启动补发

### 3. 用户体验

**提示信息**：
```
"检测到未完成的交易，已自动处理。如权限未开通，请重启应用"
```

**含义**：
- 已尝试自动处理
- 大部分情况下已成功
- 如果仍未开通，重启应用会自动补发
