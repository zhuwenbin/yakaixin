# 登录后跳转问题修复方案

## 🔍 问题分析

### 问题现象
未登录状态在题库页面购买商品后，登录成功还是跳转到首页，而不是返回题库页面。

### 根本原因

**路由拦截器的执行时机问题**：

1. 用户在商品详情页点击购买（未登录）
   - 跳转到登录页，传递 `returnPath: AppRoutes.mainTab`
   - `LoginCenterPage(returnPath: '/main-tab')` 被创建

2. 用户输入账号密码，点击登录
   - `authProvider.loginWithSms()` 执行成功
   - Token 被保存到本地存储
   - `authProvider.isLoggedIn` 变为 `true`

3. **路由拦截器立即被触发**（在 `login_page.dart` 的 `Future.microtask` 之前）
   - 检测到 `hasToken == true` 且 `isOnLoginPage == true`
   - 尝试从 `state.extra` 获取 `returnPath`
   - **但是 `state.extra` 可能已经丢失或为空**
   - 因为没有 `returnPath`，强制返回 `AppRoutes.mainTab`

4. `login_page.dart` 中的跳转逻辑执行（但已经晚了）
   - `Future.microtask()` 中的 `context.go(returnPath)` 执行
   - 但此时路由拦截器已经强制跳转到 `mainTab` 了

### 核心问题

**路由拦截器触发时，`state.extra` 可能已经丢失**，导致无法获取 `returnPath`。

## ✅ 解决方案

### 方案：使用全局 Provider 保存 returnPath

创建一个全局 Provider 来保存 `returnPath`，确保路由拦截器能够获取到：

1. **创建 `loginReturnPathProvider`**
   - 用于保存登录后的返回路径
   - 在路由拦截器中优先从 Provider 获取

2. **在 `LoginCenterPage` 构建时保存 returnPath**
   - 当页面构建时，立即保存 `returnPath` 到 Provider
   - 确保路由拦截器能够获取到

3. **在路由拦截器中优先从 Provider 获取**
   - 优先从 `loginReturnPathProvider` 获取
   - 如果 Provider 中没有，再从 `state.extra` 获取
   - 使用后清除 Provider 中的值

## 📝 实现细节

### 1. 创建 Provider

```dart
// lib/core/utils/login_return_path_provider.dart
final loginReturnPathProvider = StateProvider<String?>((ref) => null);

void setLoginReturnPath(WidgetRef ref, String? returnPath) {
  ref.read(loginReturnPathProvider.notifier).state = returnPath;
}
```

### 2. 在 LoginCenterPage 中保存

```dart
// lib/features/auth/views/login_center_page.dart
@override
Widget build(BuildContext context, WidgetRef ref) {
  // ✅ 立即保存返回路径到 Provider
  if (returnPath != null && returnPath!.isNotEmpty) {
    Future.microtask(() {
      setLoginReturnPath(ref, returnPath);
    });
  }
  
  return LoginPage(returnPath: returnPath);
}
```

### 3. 在路由拦截器中获取

```dart
// lib/app/routes/app_router.dart
if (hasToken && isOnLoginPage) {
  // ✅ 优先从 Provider 中获取返回路径
  final returnPathFromProvider = ref.read(loginReturnPathProvider);
  
  // ✅ 如果 Provider 中没有，尝试从路由 state 中获取
  final extra = state.extra as Map<String, dynamic>?;
  final returnPathFromExtra = extra?['returnPath'] as String?;
  
  final returnPath = returnPathFromProvider ?? returnPathFromExtra;
  
  if (returnPath != null && returnPath.isNotEmpty) {
    // ✅ 清除 Provider 中的返回路径
    if (returnPathFromProvider != null) {
      ref.read(loginReturnPathProvider.notifier).state = null;
    }
    return returnPath;
  } else {
    return AppRoutes.mainTab;
  }
}
```

## 🎯 工作流程

1. 用户在商品详情页点击购买（未登录）
2. 跳转到登录页，传递 `returnPath: AppRoutes.mainTab`
3. `LoginCenterPage` 构建时，保存 `returnPath` 到 Provider
4. 用户输入账号密码，点击登录
5. 登录成功，Token 保存
6. **路由拦截器触发**，从 Provider 获取 `returnPath`
7. 使用 `returnPath` 跳转，返回题库页面
