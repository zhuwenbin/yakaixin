## ❌ 错误情况日志

### 错误1: 未找到产品

```
❌ 未找到产品: com.yakaixin.yakaixin.123
💡 请检查:
   1. App Store Connect中是否已创建该产品
   2. 产品状态是否为"等待审核"或"已批准"
   3. Bundle ID是否匹配
```

**解决方案**:
1. 登录 App Store Connect
2. 检查产品是否存在
3. 确认产品状态
4. 检查 Bundle ID 是否匹配

---

### 错误2: 订单信息不完整

```
❌ 错误：订单信息不完整
   缺失字段:
   - 订单ID
   - 商品ID
   操作: 完成交易（避免重复购买）
```

**原因**:
- 内存中的订单信息被清空
- 回调时找不到对应的订单

**解决方案**:
- 确保在 `purchase()` 方法中正确保存订单信息
- 检查是否有其他地方清空了变量

---

### 错误3: 获取收据失败

```
❌ 获取收据失败
   返回值: null
   操作: 完成交易（避免重复购买）
```

**原因**:
- App Store 收据不存在
- 设备没有登录 Apple ID

**解决方案**:
1. 确保设备已登录 Apple ID
2. 确保在沙盒账号中登录
3. 重新发起购买

---

### 错误4: 后端验证失败

```
❌ ========== 验证收据失败 ==========
   错误类型: Exception
   错误信息: 网络请求失败
   发生位置: _verifyAndFinishTransaction

   操作: 完成交易（避免重复购买）
   说明: 即使验证失败也要通知Apple完成交易
   原因: 如果不完成，用户会一直收到重复购买提示
   ✅ 交易已完成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**原因**:
- 网络连接失败
- 后端接口异常
- 收据格式错误

**解决方案**:
1. 检查网络连接
2. 检查后端接口状态
3. 查看后端日志

## 📊 日志分析技巧

### 1. 快速定位问题

搜索关键字：
- `❌` - 查找所有错误
- `⚠️` - 查找所有警告
- `步骤` - 查看流程执行到哪一步
- `购买状态` - 查看支付状态

### 2. 验证流程完整性

检查是否包含以下日志：
- [ ] `步骤0: 接收购买参数`
- [ ] `步骤1: 查询产品信息`
- [ ] `步骤2: 发起购买请求`
- [ ] `购买更新回调`
- [ ] `步骤3: 验证收据流程`
- [ ] `步骤4: 调用后端验证`
- [ ] `步骤5: 完成交易`
- [ ] `内购流程完成`

### 3. 关键数据检查

确认以下数据正确：
- [ ] 产品ID格式: `com.yakaixin.yakaixin.{goods_id}`
- [ ] 产品价格与后台一致
- [ ] 交易ID不为空
- [ ] 收据数据长度 > 0
