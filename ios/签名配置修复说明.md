# Xcode 自动管理签名配置修复说明

## ❌ 问题原因

### 错误信息
```
Runner has conflicting provisioning settings. 
Runner is automatically signed for development, 
but a conflicting code signing identity Apple Distribution has been manually specified.
```

### 问题分析

当使用 **Xcode 自动管理签名**（`CODE_SIGN_STYLE = Automatic`）时：

1. ✅ Xcode 会自动根据配置选择证书类型
   - Debug 配置 → 自动使用 Development 证书
   - Release 配置 → 自动使用 Distribution 证书

2. ❌ **不能手动指定** `CODE_SIGN_IDENTITY`
   - 如果手动指定了 `CODE_SIGN_IDENTITY = "Apple Distribution"`
   - 会与自动管理签名冲突
   - Xcode 无法正确处理

---

## ✅ 解决方案

### 修复方法：删除手动指定的 CODE_SIGN_IDENTITY

**修改前：**
```xml
CODE_SIGN_ENTITLEMENTS = Runner/Runner.entitlements;
CODE_SIGN_IDENTITY = "Apple Distribution";  // ❌ 手动指定，会冲突
CODE_SIGN_STYLE = Automatic;
```

**修改后：**
```xml
CODE_SIGN_ENTITLEMENTS = Runner/Runner.entitlements;
// CODE_SIGN_IDENTITY 已删除，让 Xcode 自动选择 ✅
CODE_SIGN_STYLE = Automatic;
```

### 为什么这样修复？

1. ✅ **自动管理签名的原则**：
   - 当 `CODE_SIGN_STYLE = Automatic` 时
   - Xcode 会根据配置自动选择证书类型
   - 不需要手动指定 `CODE_SIGN_IDENTITY`

2. ✅ **Xcode 的自动选择规则**：
   - Release 配置 → 自动使用 Distribution 证书（如果存在）
   - Debug 配置 → 自动使用 Development 证书

3. ✅ **前提条件**：
   - 钥匙串中必须有对应的证书
   - 已勾选 "Automatically manage signing"
   - Team 已正确配置

## 📋 正确的配置方式

### 使用 Xcode 自动管理签名时

#### ✅ 正确配置

```xml
CODE_SIGN_STYLE = Automatic;           // ✅ 启用自动管理
// 不设置 CODE_SIGN_IDENTITY          // ✅ 让 Xcode 自动选择
DEVELOPMENT_TEAM = 6LPT9NQFKF;        // ✅ 指定团队
```

#### ❌ 错误配置

```xml
CODE_SIGN_STYLE = Automatic;
CODE_SIGN_IDENTITY = "Apple Distribution";  // ❌ 会冲突
```

### 在 Xcode GUI 中配置

1. **打开 Xcode**
   ```bash
   open ios/Runner.xcworkspace
   ```

2. **配置签名**
   - 选择项目 → Runner target → **Signing & Capabilities**
   - ✅ 勾选 **"Automatically manage signing"**
   - ✅ 选择 Team：**6LPT9NQFKF (Sun Zhanpeng)**
   - ❌ **不要**手动选择 Code Signing Identity
   - ❌ **不要**手动选择 Provisioning Profile

3. **让 Xcode 自动选择**
   - Xcode 会根据配置自动选择证书
   - Release → Distribution 证书
   - Debug → Development 证书

## 🔄 Archive 时使用 Distribution 证书

### 确保 Archive 使用 Release 配置

1. **在 Xcode 中**
   - Scheme → Edit Scheme...
   - 选择 **Archive**
   - Build Configuration 选择 **Release** ✅

2. **Archive 操作**
   - Product → Archive
   - Xcode 会自动使用 Distribution 证书签名

### 验证 Archive 使用的证书

Archive 完成后，在 Organizer 中：
1. 选择 Archive
2. 点击 **Distribute App**
3. 查看签名信息
4. 应该显示 **Apple Distribution** 证书
