# 应用备案修改说明

## ✅ 备案后可以修改

应用备案完成后，**可以修改**，但需要遵循相关规定和流程。

---

## 📋 可以修改的内容

### 1. 应用基本信息
- ✅ 应用名称
- ✅ 应用图标
- ✅ 应用描述
- ✅ 应用分类
- ✅ 应用版本号
- ✅ 开发者信息（部分）

### 2. 签名证书信息 ⚠️
- ✅ **可以修改**，但需要重新提交审核
- ✅ 更换签名证书（如从开发证书改为Distribution证书）
- ✅ 更新证书指纹和公钥

### 3. 域名信息
- ✅ 可以添加或修改使用的域名
- ⚠️ 需要重新审核

### 4. 平台信息
- ✅ 可以添加新的运行平台（如新增iOS或Android）
- ✅ 可以修改平台特征信息

---

## ⚠️ 修改限制和注意事项

### 1. 签名证书修改
- **可以修改**：从开发证书改为Distribution证书
- **需要操作**：
  1. 在备案系统中提交修改申请
  2. 上传新的证书信息（SHA-1指纹、公钥）
  3. 等待审核通过
- **建议**：
  - 如果当前使用开发证书，建议尽快更换为Distribution证书
  - 更换后需要更新备案信息

### 2. Bundle ID / Package Name
- ⚠️ **通常不能修改**（这是应用唯一标识）
- 如需修改，可能需要重新备案或创建新应用备案

### 3. 开发者主体信息
- ⚠️ **部分信息不能修改**（如企业名称、统一社会信用代码）
- ✅ 可以修改联系方式、地址等

---

## 🔄 修改流程

### 步骤1：登录备案系统
1. 登录应用备案管理系统
2. 找到已备案的应用

### 步骤2：提交修改申请
1. 点击"修改"或"变更"按钮
2. 选择需要修改的内容
3. 填写新的信息

### 步骤3：上传证明材料
- 新的签名证书信息
- 相关证明材料（如需要）

### 步骤4：等待审核
- 通常审核时间为 1-3 个工作日
- 审核通过后，修改生效

---

## 📝 从开发证书改为Distribution证书的步骤

### 1. 获取Distribution证书信息

#### 方法1：通过Xcode自动创建
```bash
# 1. 打开Xcode
open ios/Runner.xcworkspace

# 2. 选择项目 → Runner target → Signing & Capabilities
# 3. 确保 Release 配置使用 Distribution 证书
# 4. 构建 Release 版本
flutter build ios --release

# 5. 获取证书信息
security find-identity -v -p codesigning | grep -i "distribution"
```

#### 方法2：通过Apple Developer Portal
1. 登录 [Apple Developer Portal](https://developer.apple.com/account/)
2. 创建 Apple Distribution 证书
3. 安装到钥匙串
4. 使用脚本获取信息：
```bash
cd ios
./获取Distribution证书信息.sh
```

### 2. 更新备案信息
1. 登录备案系统
2. 找到iOS平台配置
3. 更新以下信息：
   - 公钥（新的Distribution证书公钥）
   - 证书SHA-1指纹（新的Distribution证书指纹）
4. 提交审核

## ⚠️ 重要提醒

### 1. 证书更换时机
- ✅ **建议尽快更换**：如果当前使用开发证书，建议在备案通过后尽快更换为Distribution证书
- ✅ **不影响已备案应用**：更换证书不会影响已备案的应用，但需要更新备案信息

### 2. 证书有效期
- ⚠️ 开发证书通常有效期1年
- ⚠️ Distribution证书通常有效期3年
- ⚠️ 证书过期前需要续期并更新备案信息

### 3. 多证书管理
- ✅ 可以同时备案多个证书（正式签名和备用签名）
- ✅ 建议同时提交正式签名和备用签名信息

## 📞 常见问题

### Q1: 备案后修改需要多长时间？
**A:** 通常1-3个工作日，具体时间取决于审核进度。

### Q2: 修改期间应用会受影响吗？
**A:** 不会。修改备案信息不会影响已上架的应用运行。

### Q3: 可以同时备案多个证书吗？
**A:** 可以。建议同时提交正式签名和备用签名。

### Q4: 证书过期了怎么办？
**A:** 需要在证书过期前续期，并更新备案信息中的证书指纹和公钥。

### Q5: 从开发证书改为Distribution证书必须修改吗？
**A:** 不是必须的，但强烈建议：
- Distribution证书有效期更长（3年 vs 1年）
- Distribution证书是正式发布的标准证书
- 部分应用商店要求使用Distribution证书
